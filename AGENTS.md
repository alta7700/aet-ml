# AGENTS.md — Правила ведения проекта

Этот файл описывает конвенции для AI-агентов и людей, работающих с проектом.
Правила обязательны: не нарушай их без явного согласования с владельцем проекта.

---

## 1. Версионирование скриптов

### Правило

Каждая воспроизводимая попытка / эксперимент / модель — **отдельный замороженный файл**.
Файлы начинаются с префикса `v{NNNN}_`, где NNNN — четырёхзначный порядковый номер.

```
scripts/
  v0001_train_elasticnet.py
  v0002_train_temporal_lags_kalman.py
  v0003_train_shap_conformal_elasticnet.py
  v0004_train_ridge_huber.py
  v0005_train_shap_conformal_ridge_huber.py
  ...
```

Просматривая папку `scripts/` в алфавитном порядке, видна полная история экспериментов.

### Когда создавать новый версионированный файл

- Новая модель или набор гиперпараметров → новый файл
- Новый тип эксперимента (temporal, conformal, SHAP) → новый файл
- Любое изменение, результат которого нужно будет воспроизвести → новый файл

### Когда НЕ создавать новый файл

- Эксперимент дал явно плохой результат и не будет использоваться → можно менять on the fly
- Технический фикс (SyntaxWarning, PerformanceWarning) без изменения логики → правь текущий
- Изменения в `build_dataset_*.py` — они не обучают модель, версионируются через git

### Процесс создания следующей версии

```bash
# Копируй предыдущую как базу, не пиши с нуля
cp scripts/v0004_train_ridge_huber.py scripts/v0005_train_something.py
# Правь копию
```

### Правила заморозки

Версионированный файл (`v{NNNN}_*.py`) **нельзя редактировать** после того как он
запущен и результаты зафиксированы в `RESULTS.md`. Он — снимок того, что дало конкретный
результат. Если нужно что-то исправить — создавай следующую версию.

## 2. Каталог результатов

Каждый версионированный скрипт пишет результаты в свою папку:

```
results/
  v0001/   ← выход v0001_train_elasticnet.py
  v0002/   ← выход v0002_train_temporal_lags_kalman.py
  v0003/   ← выход v0003_train_shap_conformal_elasticnet.py
  v0004/   ← выход v0004_train_ridge_huber.py
  v0005/   ← выход v0005_train_shap_conformal_ridge_huber.py
```

Внутри скрипта:
```python
OUT_DIR = _ROOT / "results" / "v0001"  # менять номер на свой
```

Старые папки `results/best_models/`, `results/temporal/`, `results/shap_conformal/`
остаются как есть — это legacy от скриптов до введения конвенции.

## 3. Заголовок каждого версионированного скрипта

Первый докстринг должен содержать:

```python
"""v{NNNN} — Краткое название эксперимента

Версия:    v{NNNN}
Дата:      YYYY-MM-DD
Предыдущая версия: v{NNNN-1}_имя_файла.py  (или "нет" для v0001)
Результаты: results/v{NNNN}/

Что изменено по сравнению с предыдущей версией:
  - ...

Гиперпараметры:
  Модель LT2: Ridge(alpha=100, fit_intercept=True, solver="auto")  [детерминирован]
  Модель LT1: HuberRegressor(epsilon=1.35, alpha=0.0001, max_iter=500)  [детерминирован]
  Imputer:    SimpleImputer(strategy="median")
  Scaler:     StandardScaler(with_mean=True, with_std=True)
  ElasticNet: random_state=42  [если используется]
  RandomForest/LGBM: random_state=42  [если используется]

Ожидаемые результаты:
  LT2: MAE ≈ X.XXX мин, R² ≈ X.XXX, ρ ≈ X.XXX
  LT1: MAE ≈ X.XXX мин, R² ≈ X.XXX, ρ ≈ X.XXX

Воспроизведение:
  uv run python scripts/v{NNNN}_имя_файла.py
  uv run python scripts/v{NNNN}_имя_файла.py --no-plots
"""
```

---

## 4. Ведение RESULTS.md

- **Новые эксперименты добавляются в конец раздела «Лог»** как запись `v{NNNN}`.
- **Старые записи не удаляются и не редактируются** — только помечаются `[superseded by vNNNN]`.
- Каждая запись ссылается на версионированный скрипт: `scripts/v{NNNN}_*.py`.
- Числа в RESULTS.md берутся из фактического вывода скрипта (не приблизительно).
- Перед фиксацией результатов: **запусти скрипт и убедись что он работает без ошибок**.

---

## 5. Воспроизведение

Чтобы воспроизвести любой исторический результат:

```bash
uv run python scripts/v0001_train_elasticnet.py --no-plots
# или любой другой номер
```

Каждый скрипт самодостаточен: содержит все гиперпараметры внутри себя,
не зависит от состояния других скриптов.

---

## 6. Нечисловые скрипты (build_dataset, visualize и т.д.)

Скрипты типа `build_dataset_*.py`, `visualize_*.py`, `extract_*.py` **не версионируются**
по этой схеме — они не обучают модели и результат не зависит от случайности.
Их история отслеживается через git.
