# AGENTS.md — Конвенции проекта

Описывает правила для AI-агентов и людей, работающих с проектом.
Правила обязательны: не нарушай их без явного согласования с владельцем проекта.

---

## 1. Структура проекта

```
scripts/
  v{NNNN}_*.py        — версионированные скрипты экспериментов
  eval_utils.py       — метрики и графики (не версионируется)
  evaluate_all.py     — прогон всех версий, генерация артефактов (не версионируется)
  build_dataset_*.py  — сборка датасета (не версионируется)

results/
  common.md           — методология, метрики, датасет, ограничения
  v{NNNN}.md          — описание каждого эксперимента
  v{NNNN}/            — выходные данные скрипта (CSV, PNG)

artefacts/
  v{NNNN}/            — acc_by_time.png, tde.png (из evaluate_all.py)
  comparison_lt2.png
  comparison_lt1.png

dataset/
  merged_features_ml.parquet   — основной датасет ML
  session_params.parquet        — параметры сессий
```

## 2. Версионирование скриптов

### Принцип

Каждый воспроизводимый эксперимент — **отдельный замороженный файл** с префиксом `v{NNNN}_`, где NNNN — четырёхзначный порядковый номер. Папка `scripts/` в алфавитном порядке читается как хронология экспериментов.

### Когда создавать новую версию

- Новая модель, алгоритм или набор гиперпараметров
- Новый тип feature engineering или постобработки
- Любое изменение, результат которого нужно воспроизвести

### Когда не создавать

- Технический фикс (предупреждение, опечатка) без изменения логики — правь текущий файл
- Скрипты `build_dataset_*.py`, `eval_utils.py`, `evaluate_all.py` — не версионируются по этой схеме

### Создание следующей версии

```bash
# Копируй предыдущую как базу, не пиши с нуля
cp scripts/v0005_train_something.py scripts/v0006_train_next.py
# Правь копию
```

### Заморозка

Версионированный файл **нельзя редактировать** после того как он запущен и описание зафиксировано в `results/v{NNNN}.md`. Файл — снимок того, что дало конкретный результат. Для исправлений — создавай следующую версию.

## 3. Заголовок версионированного скрипта

Первый докстринг содержит:

```python
"""v{NNNN} — Краткое название эксперимента

Версия:    v{NNNN}
Дата:      YYYY-MM-DD
Предыдущая версия: v{NNNN-1}_имя_файла.py  (или "нет" для первой версии)
Результаты: results/v{NNNN}/

Что изменено:
  - ...

Гиперпараметры:
  Модель LT2: Ridge(alpha=100, fit_intercept=True, solver="auto")  [детерминирован]
  Модель LT1: HuberRegressor(epsilon=1.35, alpha=0.0001, max_iter=500)  [детерминирован]
  Imputer:    SimpleImputer(strategy="median")
  Scaler:     StandardScaler(with_mean=True, with_std=True)
  ElasticNet: random_state=42  [если используется]

Ожидаемые результаты:
  LT2: MAE ≈ X.XXX мин, R² ≈ X.XXX, ρ ≈ X.XXX
  LT1: MAE ≈ X.XXX мин, R² ≈ X.XXX, ρ ≈ X.XXX

Воспроизведение:
  uv run python scripts/v{NNNN}_имя_файла.py
  uv run python scripts/v{NNNN}_имя_файла.py --no-plots
"""
```

```python
OUT_DIR = _ROOT / "results" / "v0006"  # менять номер на свой
```

## 5. Документация эксперимента

На каждый версионированный скрипт — файл `results/v{NNNN}.md`.

### Структура файла

```markdown
# v{NNNN} — Название

**Скрипт:** scripts/v{NNNN}_*.py
**Дата:** YYYY-MM-DD
**Предыдущая версия:** v{NNNN-1}
**Данные:** results/v{NNNN}/
**Артефакты:** artefacts/v{NNNN}/

## Что делает
[мотивация и описание эксперимента]

## Гиперпараметры
[полная таблица, включая random_state]

## Признаки
[какие feature-группы и почему]

## Результаты
[MAE, R², ρ, Acc@δ, TDE — числа из фактического вывода скрипта]

## Выводы
[что получилось, физиологическая интерпретация, что делать дальше]
```

### Правила

- Файл создаётся после успешного прогона скрипта
- Все числа берутся из фактического вывода — не приблизительно
- Файл не редактируется после фиксации (скрипт заморожен — описание тоже)
- Если версия вытеснена более новой: добавить `> Superseded: v{NNNN}` в начало

## 6. Артефакты (графики оценки)

Графики `Acc@δ(t_norm)` и `TDE` генерируются через `evaluate_all.py`:

```bash
uv run python scripts/evaluate_all.py          # все версии, LT1 и LT2
uv run python scripts/evaluate_all.py --target lt2 --versions v0004 v0006
```

Сохраняются в `artefacts/v{NNNN}/`:
- `lt2_acc_by_time.png` — Acc@δ в зависимости от нормализованного времени теста
- `lt2_tde.png` — ошибка детектирования порога по участникам
- `artefacts/comparison_lt2.png` — сравнение всех версий

### Добавление новой версии в evaluate_all.py

1. Написать `prep_lt2_v{NNNN}(df_full, sp)` и `prep_lt1_v{NNNN}(df_full, sp)` — функции подготовки данных и признаков
2. Добавить запись в словарь `VERSIONS`
3. Запустить `evaluate_all.py`

---

## 7. Воспроизведение любого эксперимента

```bash
uv run python scripts/v{NNNN}_имя_файла.py --no-plots
```

Каждый скрипт самодостаточен: содержит все гиперпараметры внутри себя, не зависит от состояния других скриптов.

## 8. Нечисловые скрипты

`build_dataset_*.py`, `eval_utils.py`, `evaluate_all.py` не версионируются по схеме `v{NNNN}` — их история отслеживается через git.
