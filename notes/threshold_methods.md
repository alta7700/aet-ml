# Методологические решения по определению порогов LT1 и LT2

## LT2 — время наступления

**Используется:** `target_time_to_lt2_center_sec` (из `targets.parquet`)  
**Источник:** `lt2_time_center_sec` → восстановлено из `lt2_power_w` через интерполяцию power→time

### Метод: modified Dmax на кубическом сплайне (moddmax_cubic)

Кубический сплайн строится в плоскости **мощность–лактат** (не время–лактат).  
LT2 = точка максимального перпендикулярного расстояния от прямой (стартовая точка → последняя точка).

**Почему кубический сплайн, а не PCHIP:**
- Dmax — геометрический метод, требует гладкой кривой с непрерывной второй производной (C²)
- PCHIP гарантирует монотонность и не осциллирует, но первая производная на стыках скачет (C¹) → форма "локтя" кривой определяется менее точно
- Для поиска точки максимального расстояния важна геометрическая форма кривой, а не предотвращение осцилляций
- PCHIP используется как **контрольная проверка устойчивости**: если `lt2_pchip_delta_power_w > 25 Вт` → quality=low

**Почему мощность, а не время:**
- В ступенчатом тесте управляемая переменная — нагрузка (мощность), а не время
- Забор крови происходит с процедурной задержкой → время менее надёжно
- Подробнее: см. `LT2_METHOD.md` (шаги 1–7)

**Уточнение времени (refined):**  
Для 5 субъектов (S008, S010, S011, S013, S014) время дополнительно уточнено через  
согласование modDmax с DFA-α1=0.50 и/или HHb breakpoint.  
Используется только при `lt2_refined_valid=1` и разбросе ≤60 сек.

## LT1 — время наступления

**Используется:** `lt1_pchip_time_sec` (из `lt1_labels.parquet`)  
**НЕ используется:** `lt1_time_sec` (дискретный момент забора крови)

### Метод: baseline + 0.4 ммоль/л с PCHIP-интерполяцией пересечения

**Реализация:** `scripts/extract_lt1_labels.py`, функция `build_baseline_lt1()`  
Константа: `BASELINE_DELTA_MMOL = 0.4`

1. Baseline = среднее первых `BASELINE_N_POINTS` лактатных точек (покой/разминка)
2. Порог = baseline + 0.4 ммоль/л
3. По дискретным точкам строится PCHIP-кривая лактат→время
4. `lt1_pchip_time_sec` = точное время пересечения PCHIP-кривой с порогом (интерполяция внутри ступени)

**Почему PCHIP, а не дискретный lt1_time_sec:**
- Забор крови привязан к ступеням (каждые 3–5 мин) — точка фиксируется в момент забора
- Если лактат резко вырос, реальный порог мог наступить в середине предыдущей ступени
- `lt1_pchip_time_sec` интерполирует внутрь интервала → смещение достигает 0.9–4.3 мин
- Дискретный `lt1_time_sec` систематически запаздывает при резком росте лактата

**Почему PCHIP, а не кубический сплайн:**
- Для LT1 ищется первое пересечение порога, а не геометрический максимум расстояния
- Кубический сплайн может осциллировать между редкими точками (3–5 мин интервалы) → ложные пересечения
- PCHIP монотонен внутри каждого интервала → гарантированно не создаёт ложных порогов
- При поиске первого роста лактата монотонность важнее гладкости

**Субъекты с низким качеством LT1:**  
S015, S005, S001, S012 — `lt1_equals_lt2=1`, `quality=low`.  
У этих субъектов LT1 и LT2 совпали или не разделились алгоритмом.  
При анализе LT1-таргета эти субъекты помечаются как ненадёжные.

## Классификация тренированности субъектов

**Порог:** LT2 > 13.0 мин → trained=1

| Группа | N | Субъекты | LT2-диапазон |
|--------|---|----------|-------------|
| Нетренированные | 13 | S001, S002, S004, S005, S006, S007, S008, S011, S012, S014, S015, S016, S018 | 4.6–11.9 мин |
| Тренированные | 6 | S003, S009, S010, S013, S017, S019 | 14.3–23.6 мин |

**Почему 13 мин:**
- Данные имеют три естественных кластера: ≤11.9 мин, 14.3 мин, 16.3–23.6 мин
- Разрыв между S007 (11.9 мин) и S017 (14.3 мин) — ~2.4 мин — наибольший в средней зоне
- Порог 13 мин попадает в этот разрыв и даёт чистое разделение без пограничных случаев
- Медианный порог (11.3 мин) давал смешанную среднюю группу (S002, S008, S014, S007 — неоднородную)

**Ограничение:**  
Классификация основана только на времени LT2 без учёта VO₂max, спортивного стажа  
или биоимпеданса. Используется как прокси-метка для gating в v0108 MoE.

| Порог | Переменная | Метод | Причина выбора |
|-------|-----------|-------|---------------|
| LT2-время | `lt2_time_center_sec` | modDmax + cubic spline по power | Геометрический метод требует C², мощность надёжнее времени |
| LT2-уточнение | `lt2_refined_time_sec` | Медиана DFA-α1 + HHb + modDmax | Только при valid=1, spread≤60с |
| LT1-время | `lt1_pchip_time_sec` | baseline+0.4 ммоль/л, PCHIP-интерполяция пересечения | Монотонность важна, дискретный запаздывает на 0.9–4.3 мин |
| Тренированность | `lt2_min > 13` | Порог по временному разрыву в данных | Чистый разрыв в распределении, нет пограничных случаев |
