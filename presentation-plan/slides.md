## Слайд 2 — Проблема и актуальность

**Цель:** обозначить gap в литературе.

**Тезисы:**
- LT1 и LT2 — ключевые точки для управления тренировкой и функциональной диагностики.
- Эталонные методы (капиллярный лактат, газоанализ) инвазивны / стационарны.
- Неинвазивные маркеры (DFA-α1, breakpoint SmO₂, ЭМГ-порог) валидированы по отдельности.
- В литературе нет ни одной системы, объединяющей ЭМГ + NIRS + ВСР в единый онлайн-алгоритм одновременного предсказания обоих порогов (output.md §1.6.4, [Kimoto 2023]).

## Слайд 4 — Дизайн эксперимента

**Цель слайда:** показать протокол и аппаратную сторону одной картой.

**Протокол (output.md §2.2):**
- Велоэргометр Lode Excalibur Sport (электромагнитное управление нагрузкой).
- Разминка: 30 Вт, 1 мин.
- Ступенчатый тест: +30 Вт каждые 3 мин, старт 60 Вт, до волевого отказа.
- Каденс фиксированный 65–80 об/мин; критерий остановки — невозможность поддерживать каденс.
- Восстановление 2–4 мин при 60–100 Вт (также записывается).

**Капиллярный лактат:**
- За 15 c до начала каждой новой ступени — забор из пальца.
- Анализатор Biosen C-line, диапазон 0,5–40 ммоль/л, погрешность <2 %.

**Разметка ground truth (output.md §2.6):**
- LT1 — момент пересечения «baseline + 0,4 ммоль/л» на PCHIP-интерполяции лактата в плоскости мощность–лактат.
- LT2 — модифицированный Dmax (ModDmax) на кубическом сплайне.

**Аппаратный комплекс (output.md §2.3):**

| Сигнал | Устройство | Точка наложения | Частота |
|---|---|---|---|
| ЭМГ + ИМУ (×2) | Delsys Trigno Avanti | VL дистально + проксимально (SENIAM) | 1926 Гц / 148 Гц |
| NIRS (SmO₂ + O₂Hb/HHb/THb/HBDiff) | Train.Red PLUS | брюшко VL | 1–4 Гц |
| RR-интервалы | Zephyr BioHarness 3 | грудной ремень | событийный |

**Визуально:** схема протокола (timeline по мощности со ступенями и точками забора лактата) + анатомическая схема расположения датчиков на ноге / груди.

## Слайд 5 — Sensync (авторская разработка)

**Цель слайда:** показать, что протокол записи реализован на собственной программной системе, и описать её roadmap.

**Мотивация (output.md §2.4.1):**
- Готовых решений для одновременной записи **1926 Гц + 148 Гц + 1–4 Гц + событийного RR** в единый файл с общей временной осью и воспроизведением — нет.
- Опыт с частично потерянными / несинхронизованными данными на предыдущем оборудовании показал критичность единой временной модели.

**Roadmap — 4 итерации:**
1. **Python + Matplotlib** — управляющий цикл трёх фаз (сбор / обработка / визуализация); ограничен в управлении.
2. **+ Flet GUI** — Python-ядро сохранено, добавлена графическая оболочка.
3. **React + Protocol Buffers** — визуализация вынесена в отдельное приложение; производительность недостаточна для ЭМГ 1926 Гц.
4. **Node.js / Electron / React (event-driven)** — текущая версия; отказ от управляющего цикла в пользу событийной шины.

**Что делает (текущая версия, §2.4.2):**
- Модульное Electron-приложение; каждый источник данных — изолированный плагин в `worker_thread` (независимая обработка, многоядерность).
- Внутренняя событийная шина — без прямых вызовов между модулями.
- Профили запуска для переключения экспериментальных сценариев без правки кода.
- Единый HDF5-файл: каждый источник — отдельный dataset, метаданные внутри того же файла, лактат добавляется отдельным каналом по завершении.
- Единая шина сессионного времени; абсолютный старт — в метаданных; разрывы соединения восстанавливаются через ресинхронизацию.

**Технологический стек (табл. 2.4.2):**

| Компонент | Технология |
|---|---|
| Платформа | Electron (Node.js 20, TypeScript) |
| UI | React 19 + ECharts |
| Многопоточность | worker_threads |
| Хранение | HDF5 (h5wasm) |
| Delsys Trigno | TCP-сокет → Trigno Control Utility |
| Train.Red PLUS | ANT+ (ant-plus) |
| Zephyr BioHarness | BLE (@abandonware/noble) |
| Обмен с воркерами | Protocol Buffers |
| Алгоритмы ЭМГ | Python compute-worker |

**Визуально:** roadmap из 4 шагов вдоль таймлайна (Python → +Flet → +React/PB → Electron/event-driven), под ним — два прямоугольника-плейсхолдера 16:10 под скриншоты UI (пользователь приложит изображения сам). Стек — компактным боковым блоком или таблицей.

## Слайд 7 — Методология

**Цель:** показать методологическую строгость на малой выборке.

**Тезисы:**
- **Валидация:** LOSO на 18 субъектах (n = 16–17 на отдельных нейросетевых конфигах из-за `dropna`).
- **Зоопарк моделей — 164 конфигурации:**
  - Линейное семейство: Ridge, ElasticNet, SVR, Gradient Boosting
  - 9 нейросетевых: LSTM, Attention-LSTM, TCN, Monotonic TCN, 4 вейвлет-семейства, ансамбль
- **3 представления MAE:** субъект-взвешенная медианная (основная), медиана |err| по окнам, raw window-weighted.
- **Метрики:** MAE, ρ Спирмена, R².
- **Статистика:** парный Уилкоксон + Холм; Манн–Уитни; критерий эквивалентности (граница 0,3 мин).
- **Конформные интервалы:** LOSO split-conformal, α = 0,1 и α = 0,2.

## Слайд 11 — Результат 4: эффект тренированности (LT1)

**Три факта:**
1. Per-subject MAE LT1: тренир. 5,42 vs нетренир. 1,86 мин (SVR/EMG+NIRS, Манн–Уитни p = 0,0008); устойчиво на 3 моделях (p = 0,0002…0,003). Для LT2 — нет (p = 0,35…0,68).
2. Время-разрешённый анализ: ошибка LT1 стабильна 1,5–2,8 мин до бина 240–480 c; > 480 c — 6,2–7,1 мин. 79 % окон с |err| > 7 мин — на тренированных.
3. Двухэкспертная система:

| Система | MAE_med | p vs generalist |
|---|---|---|
| generalist | 3,15 | — |
| oracle_sym | 2,91 | 0,246 |
| real_router_sym | 3,17 | 0,899 |
| oracle_asym | 2,91 | **0,034** |
| real_router_asym | **2,18** | 0,074 (тренд) |

Маршрутизатор: 66,7 % (на уровне базовой частоты класса).

**Биология:** у тренированных лимитирует доставка O₂ → LT1 размывается; у нетренированных — митохондрии → LT1 чётче.

**Графики:** `mae_by_trained_group.png` + `error_vs_time_to_threshold_lt1.png` + `systems_comparison.png`.

## Слайд 12 — Результат 5: SHAP-интерпретация

**Контраст:** концентрированный LT2 vs диффузный LT1.

- **LT2 / HRV (Ridge):** `hrv_mean_rr_ms` 43,5 % + `hrv_dfa_alpha1` 31,4 % = **75 %** важности на двух признаках.
- **LT1 / EMG+NIRS (SVR + Permutation):** ведущий `trainred_hhb_std` 5,2 %; топ-10 = 37,1 %. Доминирующего признака нет.
- Группы LT1: EMG 54,0 %, NIRS 40,3 %, кинематика 5,7 %.
- Биологический смысл: DFA-α1 = 0,5 — известный маркер HRVT2 [Rogers, Gronwald 2022]; для LT1 концентрированного маркера в литературе нет.

## Слайд 14 — Научная новизна

5 пунктов (§4.2), по одной строке:
1. Зависимость минимально достаточного набора сенсоров от типа порога (LT2 → ВСР; LT1 → ЭМГ+NIRS).
2. Зависимость «абсолютные vs трендовые» от класса модели (нейросети не нуждаются в 86 %; линейная LT2/HRV — критично, Δ = +0,54 мин, p = 0,006).
3. При n = 18 линейная регуляризованная регрессия значимо превосходит все 9 нейросетевых (90 : 0, 14/16 групп).
4. Эффект подгруппы тренированных для LT1: p = 0,0002…0,003; 79 % больших ошибок на ней.
5. Асимметричный двухэкспертный вариант значимо превосходит generalist; симметричный — нет.

## Слайд 15 — Ограничения и направления

**Ограничения (§4.4):**
- n = 18; для отдельных нейросетевых конфигов 16–17.
- Гетерогенная выборка, отклонение от целевой популяции триатлонистов.
- Подгруппа тренированных n = 6 → результаты по двум экспертам эксплоративны.
- Маршрутизатор 66,7 % (= базовая частота большего класса).
- LT1 conformal-интервалы ±7–9 мин — недостаточны для оперативного использования.

**Направления (§4.5):**
- Расширение выборки тренированных.
- Поиск концентрированного маркера LT1.
- Стратифицированная конформная калибровка.
- DeepExplainer / Integrated Gradients для нейросетей.

## Слайд 16 — Выводы

**Главный тезис:**

> Для LT2 достаточно одного ВСР-сенсора (MAE 1,90 мин; conformal ±3,49 мин); для LT1 — комбинации ЭМГ+NIRS (MAE 2,11 мин); при n = 18 ручные признаки превосходят нейросети 90:0; обнаружен и количественно охарактеризован эффект тренированности, ограничивающий предсказание LT1.

**Практические следствия (§4.3):**
1. LT2 онлайн: достаточно носимого кардиомонитора.
2. LT1 онлайн: ЭМГ + NIRS на работающей мышце.
3. Линейная модель требует индивидуальной калибровки `hrv_mean_rr_ms`; нейросетевая — нет.
