## Слайд 4 — Дизайн эксперимента

**Цель слайда:** показать протокол и аппаратную сторону одной картой.

**Обязательный текст (три блока):**

**Протокол ступенчатого теста (output.md §2.2):**
- Велоэргометр Lode Excalibur Sport (электромагнитное управление нагрузкой — мощность не зависит от каденса).
- Разминка: 30 Вт, 1 мин.
- Основной тест: **+30 Вт каждые 3 мин**, старт 60 Вт, до волевого отказа.
- Каденс фиксированный 65–80 об/мин; критерий остановки — невозможность поддерживать каденс.
- Восстановление 2–4 мин при 60–100 Вт (также записывается).

**Капиллярный лактат (ground truth):**
- За 15 c до начала каждой новой ступени — забор из пальца.
- Анализатор Biosen C-line; диапазон 0,5–40 ммоль/л; погрешность < 2 %.
- Разметка порогов: LT1 — пересечение «baseline + 0,4 ммоль/л» на PCHIP-интерполяции (мощность–лактат); LT2 — модифицированный Dmax на кубическом сплайне.

**Аппаратный комплекс (output.md §2.3):**

| Сигнал | Устройство | Точка наложения | Частота |
|---|---|---|---|
| ЭМГ + ИМУ (×2 датчика) | Delsys Trigno Avanti | VL дистально + проксимально (SENIAM) | 1926 Гц ЭМГ / 148 Гц гироскоп |
| NIRS (SmO₂, O₂Hb, HHb, THb) | Train.Red PLUS | брюшко VL | 1–4 Гц |
| RR-интервалы | Zephyr BioHarness 3 | грудной ремень | событийный поток |

**Визуальная подсказка:** левый блок — таймлайн протокола (ступеньки мощности с подписями длительностей и точками забора лактата); правый блок — анатомическая схема расположения датчиков на ноге и груди + миниатюрная таблица. Ground-truth разметка показана как две вертикальные пунктирные линии LT1 / LT2 на таймлайне.

## Слайд 5 — Sensync (авторская разработка)

**Цель слайда:** показать, что протокол записи реализован на собственной программной системе; описать её эволюцию (roadmap) и текущую функциональность.

**Обязательный текст (три блока):**

**Мотивация (output.md §2.4.1):**
- Готовых решений для одновременной записи **1926 Гц + 148 Гц + 1–4 Гц + событийного RR** в единый файл с общей временной осью — нет.
- Опыт с частично потерянными / несинхронизованными данными на предыдущем оборудовании показал критичность единой временной модели.

**Roadmap — 4 итерации:**
1. **Python + Matplotlib** — управляющий цикл трёх фаз (сбор / обработка / визуализация); ограничен в управлении.
2. **+ Flet GUI** — Python-ядро сохранено, добавлена графическая оболочка для управления записью.
3. **React + Protocol Buffers** — визуализация вынесена в отдельное приложение; производительность недостаточна для ЭМГ 1926 Гц.
4. **Node.js / Electron / React (event-driven)** — текущая версия; отказ от управляющего цикла в пользу событийной шины.

**Что делает (текущая версия, §2.4.2):**
- Модульное Electron-приложение; каждый источник данных — изолированный плагин в `worker_thread` (независимая обработка, многоядерность).
- Внутренняя событийная шина — без прямых вызовов между модулями.
- Профили запуска — переключение экспериментальных сценариев без правки кода.
- Единый HDF5-файл: каждый источник — отдельный dataset, метаданные внутри того же файла; лактат добавляется отдельным каналом по завершении теста.
- Единая шина сессионного времени; абсолютный старт — в метаданных; разрывы соединения восстанавливаются через ресинхронизацию.

**Технологический стек (табл. 2.4.2):**

| Компонент | Технология |
|---|---|
| Платформа | Electron (Node.js 20, TypeScript) |
| UI | React 19 + ECharts |
| Многопоточность | worker_threads |
| Хранение | HDF5 (h5wasm) |
| Delsys Trigno | TCP-сокет → Trigno Control Utility |
| Train.Red PLUS | ANT+ (ant-plus) |
| Zephyr BioHarness | BLE (@abandonware/noble) |
| Обмен с воркерами | Protocol Buffers |
| Алгоритмы ЭМГ | Python compute-worker |

**Визуальная подсказка / макет:**
- **Сверху слайда** — горизонтальный roadmap из 4 этапов вдоль таймлайна (Python+Matplotlib → +Flet GUI → +React/PB → Electron/event-driven). Четвёртый этап подсвечен как текущий.
- **Нижняя часть слайда — два пустых прямоугольника соотношением 16:10 рядом** под скриншоты UI (пользователь приложит изображения позже). Подписать миниатюры коротко: «UI записи» и «Структура HDF5» (либо нейтрально «Sensync UI · 1» / «Sensync UI · 2», если содержание скринов окажется другим).
- **Сбоку или внизу** — компактная таблица стека (или ключевые элементы стека одной строкой).
- **Главный визуальный акцент** — стрелка прогресса вдоль roadmap; ощущение, что 4-я версия — финальная.
