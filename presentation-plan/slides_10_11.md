## Слайд 10 — Результат 3: линейные модели против нейросетевых

**Главный тезис (заголовок-акцент):** при n = 18 ручные признаки + регуляризованная регрессия превосходят все 9 нейросетевых архитектур.

**Обязательные числа (табл. 3.4):**
- Всего парных сравнений по критерию Уилкоксона с поправкой Холма: 648
- Значимых (p_holm < 0,05): 168
- **Линейное семейство vs 9 нейросетевых: 90 значимых побед, 0 значимых поражений в 14 из 16 групп** «целевая × модальность × вариант»
  - with_abs: 39 побед, 0 поражений (6 из 8 групп)
  - noabs: 51 победа, 0 поражений (все 8 групп)
- В двух не-доминирующих группах (LT1 / NIRS, LT2 / NIRS, with_abs) — нейтрально, без значимого превосходства ни одного класса.

**Контекстный тезис:** согласуется с [Rodu и др., 2024]: при n < 50 supervised ML должен **дополнять**, а не заменять гипотетико-дедуктивный подход; ансамблевые методы предпочтительнее глубоких архитектур.

**Перечисление побеждённых архитектур (одной строкой):** LSTM, Attention-LSTM, TCN, Monotonic TCN, четыре конфигурации вейвлет-семейства, ансамбль (вейвлет-TCN + ElasticNet).

**Визуальная подсказка:** **главный визуальный акцент — крупная цифра «90 : 0»** на тёмном фоне (либо счёт в стиле спортивного табло). Под ней — две тепловые карты p_holm как доказательство (одна по LT2, одна по LT1, обе with_abs). На картах красный = строка значимо лучше столбца, синий = значимо хуже.

## Слайд 11 — Результат 4: эффект тренированности (LT1)

**Цель слайда:** показать центральную диагностическую находку работы — асимметрию ошибки между подгруппами и её решение через двухэкспертную схему.

**Структура — три связных факта:**

**Факт 1. Per-subject MAE для LT1 значимо выше на тренированных (табл. 3.9, Манн–Уитни):**
- SVR / EMG+NIRS: тренир. (n = 6) **5,42 мин** vs нетренир. (n = 12) **1,86 мин**, p = 0,0008.
- Эффект устойчив на всех трёх топовых LT1-моделях (p = 0,0002…0,003).
- **Для LT2 эффекта нет** (p = 0,35…0,68). Это специфично именно для LT1.

**Факт 2. Время-разрешённый анализ ошибки (табл. 3.10):**
- Ошибка LT1 стабильна 1,5–2,8 мин на бинах ≤ 480 c до порога.
- В бине **> 480 c до LT1 ошибка вырастает до 6,2–7,1 мин**.
- 79 % окон с |err| > 7 мин — окна тренированных (при их 44 % доле в общем числе окон).

**Факт 3. Система двух экспертов для LT1 (табл. 3.11):**

| Система | MAE_median, мин | p vs generalist |
|---|---|---|
| generalist | 3,15 | — |
| oracle_sym (симметричная) | 2,91 | 0,246 |
| real_router_sym | 3,17 | 0,899 |
| oracle_asym (асимметричная) | 2,91 | **0,034** |
| **real_router_asym** | **2,18** | 0,074 (тренд) |

- Маршрутизатор: логрег на 11 базальных признаках (HR_baseline, body_fat_pct, age, baseline SmO₂ и др.), точность LOSO 66,7 % — на уровне базовой частоты большего класса.
- Симметричный вариант (trained-специалист + untrained-специалист) **не работает** при n = 6 в подгруппе тренированных.
- Асимметричный вариант (untrained-специалист + generalist для тренированных) значимо лучше generalist (oracle) и даёт тренд при реальной маршрутизации.

**Биологическая интерпретация (одна строка):** у тренированных лимитирует доставка O₂ → LT1 размывается; у нетренированных — митохондриальная ёмкость, LT1 проявляется чётче (раздел 3.9.4; [Messonnier 2013], [Racinais 2014]).

**Визуальная подсказка:** три зоны на одном слайде:
- слева — boxplot per-subject MAE по группам тренированных/нетренированных,
- по центру — кривая ошибки по бинам времени до LT1 (со скачком в последнем бине),
- справа — bar по 5 системам с подсвеченной парой «generalist 3,15 → real_router_asym 2,18».

Если плотно — можно центральный график сделать миниатюрным, а боксплот и систему оставить крупными.
