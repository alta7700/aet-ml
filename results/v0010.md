## Механика экстремального результата LT2

### Почему σ_p=5, σ_obs=400 даёт MAE=1.708

При σ_p=5с (Q=25) и σ_obs=400с (R=160 000) Kalman-усиление стремится к нулю.

На установившемся режиме дисперсия P_ss из уравнения Риккати:

```
P_ss = (P_ss + Q) · R / (P_ss + Q + R)
P_ss ≈ √(Q · R) = √(25 · 160 000) = 2 000
K_ss = (P_ss + Q) / (P_ss + Q + R) ≈ 2 025 / 162 025 ≈ 1.3%
```

**Kalman-gain K ≈ 1.3%** означает: каждое новое предсказание модели вносит только 1.3% в обновление состояния. Фактически фильтр игнорирует модель начиная со второго окна и ведёт:

```
τ̂_t ≈ τ̂_0 − t · step
```

где τ̂_0 — первое предсказание для данного участника (в LOSO оно сделано на моделе, обученной на остальных).

### Почему это работает технически

Истинный таргет τ(t) = LT_time − t убывает строго линейно. Если первое предсказание достаточно точное, линейная экстраполяция от него даёт низкое MAE на всём горизонте. Это не свойство Kalman как метода — любой «anchor + drift» трекер дал бы то же самое.

### Почему это подгонка

1. **Параметры выбраны по тому же LOSO, на котором оцениваем.** Нет вложенной валидации для выбора σ_p и σ_obs — они подобраны под конкретные 13 участников.

2. **Вырождение к линейному трекеру.** MAE=1.708 отражает не качество ML-модели, а то, что τ(t) в принципе линейна, и первое предсказание попадает в нужный диапазон. Для произвольного нового участника гарантий нет: если первое предсказание ошибочно, весь дальнейший трек смещён.

3. **Неустойчивость по σ_obs для LT1.** Для LT1 при σ_p=5 увеличение σ_obs ухудшает результат (1.907→2.038), тогда как для LT2 улучшает — поведение разное, что указывает на зависимость от конкретной выборки, а не на физически обоснованный параметр.

## Выводы

- Формально лучший LT2 (MAE=1.708) — артефакт вырождения Kalman в линейный трекер; в диссертации не используется.
- Честный оптимум для обоих таргетов: **σ_p=15с, σ_obs=150–250с** (K≈5–15%, реальное сглаживание).
- v0009 (σ_p=15, σ_obs=150) остаётся финальным результатом линейного этапа.
- v0010 показывает, что дальнейший grid-search Kalman нецелесообразен без вложенной валидации параметров.
- Следующий шаг: сбор новых данных → ребилд датасета → нейросетевые модели (LSTM/TCN).
