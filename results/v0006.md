**Предыдущая версия:** v0004  
**Данные:** `results/v0006/`

---

## Мотивация

`hrv_mean_rr_ms` — доминирующий признак (SHAP rank #1), но несёт межсубъектный конфаунд:
- ρ_within = +0.997: внутри сессии одного спортсмена — сильный физиологический сигнал
- ρ_between = +0.753: между участниками — более тренированные имеют ниже ЧСС покоя и выше пороги

Гипотеза: нормировка к первой ступени (stage_index==0, ~60 Вт, первые 3 мин) уберёт межсубъектный offset, сохранит динамику.

---

## Что делает

**Функция `add_stage0_baselines()`:** для каждого участника вычисляет среднее каждого признака в stage_index==0 и добавляет нормированные версии:
```python
pct_change = (x - baseline) / |baseline| × 100   # для мс-признаков HRV
delta      = x - baseline                          # для безразмерных / NIRS
```

Тестирует 5 вариантов замены hrv_mean_rr_ms:

| Вариант | Описание |
|---|---|
| **ref** | Без изменений (=v0004, контроль) |
| **A** | hrv_mean_rr_ms → rr_pct_change + rr_delta |
| **B** | Все HRV мс → pct_change; + smo2_delta |
| **B2** | Все HRV мс → pct_change + delta; + smo2_delta |
| **C** | Вариант B + rr_slope_per_stage |
| **D** | Вариант A + rr_slope_per_stage |
| **E** | Только pct+scale-free, без абсолютных NIRS |

---

## Гиперпараметры

Те же что v0004 (Ridge для LT2, Huber для LT1).

**Новые производные признаки:**
- `hrv_mean_rr_ms_pct_change` = (rr_текущий − rr_stage0) / rr_stage0 × 100
- `hrv_mean_rr_ms_delta` = rr_текущий − rr_stage0
- `rr_slope_per_stage` = rr_pct_change / (stage_index + 1)
- Аналогично для hrv_sdnn_ms, hrv_rmssd_ms, hrv_sd1_ms, hrv_sd2_ms, hrv_dfa_alpha1
- `trainred_smo2_mean_delta`, `trainred_hhb_mean_delta` и др.

## Физиологическая интерпретация нормированных признаков

### Почему pct_change теряет сигнал

После нормировки к stage-0 абсолютный контекст тренированности исчезает. Вместо «RR=800 мс → тренированный спортсмен с высоким порогом» модель видит только «+25% от базового». При таком представлении все участники в схожей фазе нагрузки выглядят одинаково — LOSO-модель не может отличить тренированного от нетренированного при экстраполяции.

### Где нормировка работала бы

Нормировка к baseline полезна, когда **межсубъектный конфаунд — шум, а не сигнал**: например, калибровка к покою при детекции аритмии, где врач смотрит только на отклонение от нормы конкретного человека. В задаче предсказания порога **абсолютный уровень физиологических параметров сам по себе является частью прогностического сигнала**.

### hrv_dfa_alpha1 как альтернатива

DFA-α₁ — единственный HRV-признак, который менее зависит от базового уровня тренированности: в покое он близок к 1.0 у большинства, снижается до ~0.75 именно вблизи LT1. Это «абсолютный» физиологический порог, не требующий нормировки. В v0006 dfa_alpha1 не тестировался как замена rr_mean — перспективное направление для v0007.

---

## Анализ причины неудачи

Нормировка к stage-0 убирает межсубъектный offset, но он оказывается **сигналом, а не шумом**:

- Тренированный спортсмен: rr_stage0 ≈ 800 мс → LT2 ≈ 30 мин
- Нетренированный: rr_stage0 ≈ 600 мс → LT2 ≈ 15 мин

Абсолютный RR в любой момент теста кодирует **одновременно**:
1. Где человек находится относительно своего порога (внутрисубъектный сигнал)
2. Насколько высок у него порог (межсубъектный сигнал — «полезный конфаунд»)

После нормировки rr_pct_change у всех участников достигает похожих значений к концу теста (+20–40%), теряя межсубъектную различительную способность. LOSO видит «все спортсмены похожи в нормированном пространстве» и теряет возможность генерализации.

## Выводы

- Ни один вариант нормировки к stage-0 не улучшает результат
- Лучший нормированный вариант (A) хуже ref на +1.0 мин MAE для LT2, +1.1 мин для LT1
- ρ_between=0.753 для hrv_mean_rr_ms — полезный сигнал, не шум в задаче LOSO CV
- Для диссертации: этот эксперимент документирует, что hrv_mean_rr_ms нельзя заменить без потери точности, и обосновывает его сохранение с пометкой об ограничении
- Альтернатива, не исследованная: включение VO2max как ковариаты для partial-out. Отложено — требует дополнительных данных.
