## Что делает

Первая воспроизводимая мультимодальная модель. Обучает ElasticNet отдельно для LT2 и LT1. Перебирает несколько конфигураций признаков, выбирает лучшую по MAE в LOSO.

Session-z нормировка ЭМГ, interaction-признаки (SmO₂×RR, SmO₂×DFA, RR/Вт), running NIRS для LT1.

---

## Гиперпараметры

| Параметр | Значение |
|---|---|
| Модель | `ElasticNet(alpha=0.5, l1_ratio=0.9, max_iter=5000, tol=1e-4, fit_intercept=True, random_state=42)` |
| Imputer | `SimpleImputer(strategy="median")` |
| Scaler | `StandardScaler(with_mean=True, with_std=True)` |
| Session-z EMG | z = (x − μ_subj) / (σ_subj + 1e-8), per-subject per-feature |
| random_state | 42 (ElasticNet использует coordinate descent с рандомизацией) |

**LT2 — «NIRS+HRV+interaction+z\_EMG»:**
- NIRS: 15 признаков (smo2_mean/slope/std/drop/dt, hhb_mean/slope/std/dt, hbdiff_mean/slope/std, thb_mean/slope/std)
- HRV: 7 признаков (mean_rr, sdnn, rmssd, sd1, sd2, sd1_sd2_ratio, dfa_alpha1)
- Interaction: feat_smo2_x_rr, feat_smo2_x_dfa, feat_rr_per_watt
- Session-z EMG: 9 ключевых признаков (z_vl_dist/prox load/rest rms, mdf, mav, delta, ratio)

**LT1 — «z\_EMG+HRV»:**
- Session-z EMG: те же 9 признаков
- HRV: те же 7 признаков

## Физиологическая интерпретация

### Почему NIRS+HRV+EMG для LT2, но только EMG+HRV для LT1?

**LT2 (анаэробный порог):** совпадает с мышечным «истощением» — SmO₂ достигает «колена» кривой, HHb перегибается. NIRS напрямую отражает этот физиологический переход → высокая информативность для LT2.

**LT1 (аэробный порог):** менее выраженный NIRS-сигнал — SmO₂ ещё не достигает плато. Вместо этого LT1 коррелирует с нейромышечной усталостью (EMG: рост RMS, снижение MDF) и ранними HRV-изменениями (снижение rmssd, dfa_alpha1 → 0.75). NIRS в этом случае добавляет шум, а не сигнал.

### ElasticNet и мультиколлинеарность HRV

ElasticNet с l1_ratio=0.9 (почти Lasso) разрешает ситуацию с мультиколлинеарными hrv_sdnn_ms / hrv_sd2_ms: L1-регуляризация обнуляет один из пары, сохраняя другой. Это делает ElasticNet предпочтительным для **интерпретации LT1** по сравнению с Huber (который даёт коэффициенты ±800 из-за отсутствия L1).

### hrv_mean_rr_ms — главный признак для LT2

Доминирование RR в ElasticNet (самый высокий |коэффициент|) воспроизводится и в Ridge (v0004), и в SHAP (v0005). Это не артефакт алгоритма — признак несёт реальный прогностический сигнал. Ограничение задокументировано в `common.md`.

### S003 — элитный атлет как системный outlier

MAE LT2=5.73 мин, LT1=3.05 мин — наихудший результат. Вероятная причина: у S003 rr_stage0 значительно ниже, чем у других «тренированных» участников обучающей выборки, модель экстраполирует за пределы распределения. Для следующих версий (v0004) использован HuberRegressor, снижающий влияние S003 на обучение.

## Выводы

- ElasticNet даёт сильный baseline для обеих задач
- Для LT2 лучшая конфигурация — NIRS+HRV+interaction+z_EMG (полная мультимодальная)
- Для LT1 достаточно z_EMG+HRV — NIRS не добавляет информации для первого порога; объяснимо физиологически
- S003 — проблемный участник (элитный атлет, выброс): MAE LT2=5.73 мин, LT1=3.05 мин
- hrv_mean_rr_ms — доминирующий признак (самый высокий коэффициент по |μ|); несёт как полезный сигнал, так и межсубъектный конфаунд
- ElasticNet L1 предпочтительнее Huber для интерпретации LT1 из-за корректной обработки мультиколлинеарных HRV-признаков
