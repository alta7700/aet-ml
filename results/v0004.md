# v0004 — Ridge (LT2) + HuberRegressor (LT1)

**Скрипт:** `scripts/v0004_train_ridge_huber.py`  
**Дата:** 2026-05-07  
**Предыдущая версия:** v0001  
**Данные:** `results/v0004/`  
**Артефакты:** `artefacts/v0004/`

---

## Что делает

Заменяет ElasticNet на специализированные модели:
- **LT2 → Ridge(α=100)**: детерминирована, L2-регуляризация стабилизирует мультиколлинеарные NIRS-признаки
- **LT1 → HuberRegressor(ε=1.35)**: устойчива к выбросу S003 (элитный атлет)

Добавлены LOSO-усреднённые коэффициенты для интерпретации важности признаков.

---

## Гиперпараметры

| Параметр | LT2 | LT1 |
|---|---|---|
| Модель | `Ridge(alpha=100.0, fit_intercept=True, solver="auto")` | `HuberRegressor(epsilon=1.35, alpha=0.0001, max_iter=500, fit_intercept=True, warm_start=False)` |
| Детерминированность | да (нет случайности) | да (нет random_state) |
| Imputer | `SimpleImputer(strategy="median")` | то же |
| Scaler | `StandardScaler` | то же |
| Session-z EMG | z = (x − μ_subj) / (σ_subj + 1e-8) | то же |

### Выбор гиперпараметров

**Ridge α=100**: grid search по LOSO; α=100 минимизирует MAE для LT2. Меньшие α (ElasticNet) хуже из-за неустойчивых коэффициентов при мультиколлинеарности NIRS.

**Huber ε=1.35**: стандартное значение (1.35 соответствует 95% гауссовских наблюдений). При ε<1.35 слишком агрессивно подавляет S003; при ε>2 поведение приближается к OLS.

## Признаки (лучшая конфигурация)

**LT2 — «NIRS+HRV+interaction+z\_EMG»** (те же что v0001):
- 32 признака: NIRS (15) + HRV (7) + Interaction (3) + z_EMG (9) — не сортировано

**LT1 — «z\_EMG+HRV+running\_NIRS»**:
- z_EMG (9) + HRV (7) + Running NIRS (3: smo2_from_running_max, hhb_from_running_min, smo2_rel_drop_pct)
- Running NIRS — каузальные признаки: падение SmO₂/рост HHb от максимума в текущей сессии

## Физиологическая интерпретация топ-признаков (LT2)

### hrv_mean_rr_ms (коэф +138, rank #1)

**Физиология:** RR-интервал обратно пропорционален ЧСС. По мере нарастания нагрузки ЧСС растёт, RR убывает — монотонный тренд.

**Почему работает:** у тренированного спортсмена rr_stage0 ≈ 800 мс → порог выше и позже; у нетренированного rr_stage0 ≈ 600 мс → порог ниже. Модель использует **абсолютный** RR одновременно как внутрисубъектный сигнал (где в тесте) и межсубъектный маркер (насколько высок порог). Оба сигнала полезны в LOSO.

**Ограничение:** ρ_between = +0.753 — конфаунд тренированности. Модель «видит», что у человека с rr=800 мс обычно бывает LT2 позже, и «знает» это по обучающей выборке. При экстраполяции на нового нетренированного спортсмена с RR=800 мс по другим причинам это даст ошибку.

### trainred_hhb_std (коэф +90, rank #2) и trainred_thb_std (коэф −86, rank #3)

**Физиология:** HHb — деоксигемоглобин, THb — суммарный гемоглобин. СКО (std) внутри 30-секундного окна отражает амплитуду колебаний сигнала.

**Почему работает:** по мере нарастания нагрузки дыхание учащается и углубляется — дыхательные колебания NIRS усиливаются. Также возможен вклад педалирования (кадан-синхронные артефакты в NIRS).

**Проблема:** неясно, отражают ли std-признаки реальные физиологические переходы или кинематические артефакты. Корреляция с `cadence_mean_rpm` — смешанная без системного паттерна. Механизм нуждается в верификации (см. `common.md`).

**Разнонаправленность:** hhb_std → позже (+90), thb_std → раньше (−86). Это физиологически последовательно: рост вариабельности HHb при HHb-доминировании означает интенсивную нагрузку, но предсказывает, что до порога ещё далеко (недостаточно «плато» HHb). Снижение thb_std соответствует стабилизации кровенаполнения перед порогом.

### feat_smo2_x_rr (коэф +81, rank #4)

**Физиология:** произведение SmO₂ × RR. SmO₂ падает с нагрузкой, RR тоже падает → их произведение убывает интенсивнее каждого в отдельности.

**Потенциальная проблема:** feat_smo2_x_rr наследует конфаунд из hrv_mean_rr_ms. Если убирать rr из модели, нужно убирать и этот признак.

**Польза:** произведение может захватывать нелинейные взаимодействия (например, точку, где SmO₂ «колено» совпадает с конкретным ЧСС), которые линейный классификатор по одиночным признакам не поймает.

### trainred_thb_mean (коэф −74, rank #5)

**Физиология:** среднее THb (кровенаполнение) в окне.

**Проблема — неустойчивый суррогат времени:** внутрисубъектная корреляция с elapsed_sec варьирует от −0.866 до +0.977. У 8 из 13 участников THb растёт с нагрузкой (вазодилатация), у 4 — падает (вазоконстрикция при очень высокой нагрузке). У некоторых нет тренда вообще.

Отрицательный коэффициент (−74): рост THb предсказывает «ещё далеко до порога» — согласуется с большинством участников (вазодилатация в начале нагрузки). Но для S011/S012 этот сигнал инвертирован.

**Рекомендация для следующей версии:** попробовать `trainred_thb_slope` (наклон THb внутри окна) вместо `thb_mean` — slope физиологически более устойчив к фазовым различиям между участниками.

## Выводы

- Ridge и ElasticNet показывают схожую MAE для LT2 — L2 vs L1 не принципиально
- Huber для LT1 критически важен для точного детектирования порога: медианный TDE = 0.87 мин
- Для диссертации: v0004 — основная (финальная) модель, используется в SHAP-анализе (v0005) и как reference в v0006
- Главное ограничение: hrv_mean_rr_ms несёт ρ_between=0.753 (конфаунд тренированности); попытки его заменить привели к MAE +1 мин (см. v0006)
