# Общее описание экспериментов

## Задача

Непрерывное предсказание времени до аэробно-анаэробного перехода по физиологическим сигналам в режиме скользящего окна.

Два целевых показателя:
- **LT2** (Лактатный порог 2, анаэробный): предсказывается для 13 участников
- **LT1** (Лактатный порог 1, аэробный): предсказывается для 9 участников (у остальных LT1 совпадает с LT2 или не размечен)

Таргет — время в секундах от конца текущего окна до момента порога. Отрицательные значения — уже за порогом.

---

## Датасет

**Файл:** `dataset/merged_features_ml.parquet`  
**Окна:** 3035 скользящих окон по ~30 сек (шаг 5 сек)  
**Участники:** 13 (S001–S014, без S008)  
**Протокол:** ступенчатый тест на велоэргометре, начало ~60 Вт, прирост ~20 Вт/ступень

### Модальности и группы признаков

| Группа | Признаки | Описание |
|---|---|---|
| NIRS | `trainred_smo2_*`, `trainred_hhb_*`, `trainred_hbdiff_*`, `trainred_thb_*` | Мышечная оксигенация (SmO₂, HHb, HbDiff, THb): mean, slope, std, drop, dt |
| HRV | `hrv_mean_rr_ms`, `hrv_sdnn_ms`, `hrv_rmssd_ms`, `hrv_sd1_ms`, `hrv_sd2_ms`, `hrv_sd1_sd2_ratio`, `hrv_dfa_alpha1` | Вариабельность сердечного ритма |
| EMG | `vl_dist_*`, `vl_prox_*` | ЭМГ m. vastus lateralis (дистальный/проксимальный): RMS, MDF, MAV, WL, ZCR, вейвлеты |

| Stage-0 norm | `*_pct_change`, `*_delta` | HRV/NIRS нормированные к первой ступени (v0006) |

## Методология оценки

### Протокол валидации: LOSO CV

Leave-One-Subject-Out Cross-Validation:
- Оставляем одного участника как тест (все его ~200–350 окон)
- Обучаем на оставшихся 12 (или 8 для LT1) участниках
- Повторяем для каждого участника по очереди
- Метрики считаются по объединённым предсказаниям всех участников

**N предсказаний:** ~2700 окон (LT2) или ~2138 окон (LT1)

### Pipeline предобработки

```
X_raw → SimpleImputer(strategy="median") → StandardScaler() → model.fit / predict
```

Session-z нормировка EMG вычисляется **до** разбивки на train/test — это допустимо, так как z-параметры считаются по всей сессии участника (одинаковы для любого разбиения).

### Детекция фаз педалирования (каузальная)

Граница push/pull определяется по PCA-проекции гироскопа. Алгоритм **каузален**: пик/впадина подтверждается только когда сигнал откатился от экстремума на `prominence` (параметр из калибровочного окна). Нет `find_peaks` по всему сигналу — нет заглядывания в будущее. Задержка детекции: ~0.1–0.3 с, несущественна для 30-секундных окон.

**Уточнение onset активации ЭМГ:** два условия, выполняющиеся одновременно:
1. `envelope ≥ median_rest + 4·MAD_rest` — уровневый критерий
2. `d(envelope)/dt ≥ median_slope_rest + 2·MAD_slope_rest` — критерий нарастания

Условие 2 отсекает хвост убывающей активации предыдущего цикла, не изменяя пороговый коэффициент k=4.

**Нон-каузальное допущение:** фильтрация сигналов (`filtfilt`, нулевая фаза) выполняется офлайн. Для реального применения потребуется замена на каузальный фильтр с компенсацией задержки.

## Метрики

### Основные (по всем ~3000 окнам)

| Метрика | Формула | Интерпретация |
|---|---|---|
| **MAE** | mean \|ŷ − y\| / 60 | Средняя ошибка в минутах |
| **R²** | 1 − SS_res/SS_tot | Доля объяснённой дисперсии (0 = predict-mean, 1 = идеал) |
| **ρ (Spearman)** | rank correlation | Порядковая корреляция, нечувствительна к масштабу |

### Acc@δ — точность в полосе допуска

**Acc@δ** = доля окон, где |ŷ − y| ≤ δ

Оценивается для δ = 30 с, 1 мин, 2 мин, 3 мин.

Дополнительно: **Acc@δ(t_norm)** — то же разбитое по нормализованному времени теста:
- t_norm = elapsed / max_elapsed ∈ [0, 1]
- 10 равных бинов
- Линия тренда: полином 2-й степени
- 95% CI: биномиальное нормальное приближение p̂ ± 1.96√(p̂(1−p̂)/n)

### TDE — Threshold Detection Error

Ошибка детектирования момента порога (в минутах, 13 или 9 значений — по одному на участника).

Метод: каждое окно оценивает абсолютный момент порога как `elapsed + ŷ`. Усредняем по участнику:
```
T̂_pred(s) = mean(elapsed + ŷ)  для участника s
T_true(s)  = mean(elapsed + y)  ≈ константа (истинное время LT)
TDE(s)     = T̂_pred(s) − T_true(s)   [сек]
```
TDE > 0 → модель «опаздывает» (предсказывает порог позже).

Отчётные цифры: **mean |TDE|** и **median |TDE|** (в минутах).

## Физиологическая интерпретация групп признаков

### HRV-признаки

| Признак | Физиологический смысл | Поведение при нагрузке |
|---|---|---|
| `hrv_mean_rr_ms` | Средний RR-интервал = 1000/ЧСС | Убывает монотонно с нагрузкой; за LT2 — ускоренное снижение |
| `hrv_rmssd_ms` | Корень СКО последовательных разностей RR; мера парасимпатической активности | Падает при нарастании нагрузки; резкий спад вблизи LT1 |
| `hrv_sdnn_ms` | СКО всех RR-интервалов; суммарная вариабельность | Снижается с нагрузкой, но менее избирательно, чем rmssd |
| `hrv_sd1_ms` | Пуанкаре: краткосрочная вариабельность = rmssd/√2 | **Математически идентичен rmssd** (r=1.000, ratio=0.7071) |
| `hrv_sd2_ms` | Пуанкаре: долгосрочная вариабельность ≈ 1.39×sdnn | Практически тот же сигнал, что sdnn (r=0.998) |
| `hrv_sd1_sd2_ratio` | sd1/sd2: баланс коротко/долгосрочной вариабельности | Меняется при переходе симпатика/парасимпатика |
| `hrv_dfa_alpha1` | DFA-α₁: фрактальный показатель корреляций RR | ≈1.0 в покое; снижается до ~0.75 вблизи LT1 — специфический нелинейный маркер порога |

### NIRS-признаки

| Признак | Физиологический смысл | Поведение при нагрузке |
|---|---|---|
| `trainred_smo2_*` | SmO₂ — мышечная оксигенация (%) | Снижается с нагрузкой; «колено» вблизи LT2 |
| `trainred_hhb_*` | HHb — деоксигемоглобин (a.u.) | Нарастает с нагрузкой; излом вблизи LT2 |
| `trainred_thb_*` | THb = HHb + HbO₂ — общий гемоглобин | Отражает локальное кровенаполнение; тренд зависит от участника |
| `trainred_hbdiff_*` | HbDiff = HbO₂ − HHb — оксигенационный баланс | Падает с нагрузкой; более стабильный сигнал, чем отдельные фракции |
| `*_slope` | Наклон признака внутри окна | Скорость изменения — важнее уровня при детекции перехода |
| `*_std` | СКО внутри окна | Может отражать дыхательные колебания или артефакты движения |
| `*_drop`, `*_dt` | Специфические разностные признаки | Каузальные изменения вблизи порога |

### Running NIRS (каузальные признаки)

| Признак | Физиологический смысл |
|---|---|
| `smo2_from_running_max` | Падение SmO₂ от максимума в сессии | Рост при нарастании усталости/лактата |
| `hhb_from_running_min` | Рост HHb от минимума в сессии | Более устойчивый к межсессионным различиям |
| `smo2_rel_drop_pct` | Относительное падение SmO₂ от максимума (%) | Нормирует межсубъектные различия в базовом SmO₂ |

### EMG-признаки

| Признак | Физиологический смысл | Поведение при нагрузке |
|---|---|---|
| `z_vl_prox/dist_rms` | Z-нормированный RMS ЭМГ | Нарастает с набором усталости; сессионная нормировка убирает межсубъектный масштаб |
| `z_vl_*_mdf` | Z-нормированная медианная частота ЭМГ | MDF снижается при накоплении лактата в мышце |
| `z_vl_*_mav` | Z-нормированное среднее абсолютное значение ЭМГ | Аналогичен RMS, высококоррелирован |

## Анализ признаков: потенциальные проблемы

### Математические дубликаты (необходимо устранить)

**hrv_rmssd_ms и hrv_sd1_ms — буквально один и тот же сигнал:**
```
sd1_ms = rmssd_ms / √2    (точное математическое тождество Пуанкаре)
r(rmssd, sd1) = 1.000000, median ratio = 0.7071
```
Включение обоих в модель не добавляет информации, но увеличивает размерность и усиливает мультиколлинеарность. **Рекомендация:** удалить `hrv_sd1_ms`, оставить `hrv_rmssd_ms` (более интерпретируемый физиологически).

**hrv_sdnn_ms и hrv_sd2_ms — практически идентичны:**
```
sd2_ms ≈ 1.39 × sdnn_ms    (аппроксимация Пуанкаре)
r(sdnn, sd2) = 0.998270, median ratio = 1.3879
```
Huber-регрессор присваивает им коэффициенты ±800 — классический признак мультиколлинеарности. **Рекомендация:** удалить `hrv_sd2_ms`, оставить `hrv_sdnn_ms`.

### Возможный временно́й прокси (осторожно)

**trainred_thb_mean — неустойчивый суррогат elapsed_sec:**

Корреляция THb с elapsed_sec внутри субъекта:

| S001 | S002 | S003 | S004 | S005 | S006 | S007 | S009 | S010 | S011 | S012 | S013 | S014 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
| +0.591 | −0.247 | +0.642 | +0.779 | +0.977 | +0.783 | +0.953 | +0.929 | +0.870 | −0.812 | −0.866 | +0.956 | −0.011 |

**Mean = +0.427, range [−0.866, +0.977]**

У 8 из 13 участников THb растёт с нагрузкой (увеличение кровенаполнения), у 4 — убывает (снижение из-за вазоконстрикции). У S002 и S014 практически нет тренда. Это означает, что `thb_mean` несёт разные физиологические сигналы у разных участников, но при LOSO модель может «видеть» его как прокси времени для конкретного участника.

**Гипотеза для следующей версии:** заменить `trainred_thb_mean` на `trainred_thb_slope` (наклон внутри окна) — slope физиологически более устойчив, не коррелирует с фазой теста так же жёстко.

### Унаследованный конфаунд в interaction-признаках

**feat_smo2_x_rr** и **feat_rr_per_watt** — оба содержат `hrv_mean_rr_ms` как сомножитель.

Если `hrv_mean_rr_ms` несёт ρ_between = +0.753 (конфаунд тренированности), то и эти признаки наследуют этот конфаунд в умноженном виде. При попытке убрать hrv_mean_rr_ms нужно также убрать эти производные.

**Потенциальная замена:** `feat_smo2_x_dfa` (SmO₂ × DFA-α₁) — оба компонента специфичны для порога, dfa_alpha1 не несёт межсубъектного конфаунда тренированности в той же мере.

### Недооценённый признак

**hrv_dfa_alpha1** занимает нижние позиции по SHAP-важности, хотя физиологически является наиболее специфичным маркером LT1:
- DFA-α₁ снижается до ~0.75 именно вблизи LT1 (граница между постоянным и некоррелированным режимом)
- Менее загрязнён межсубъектным конфаундом тренированности
- В LT1-задаче мог бы быть усилен Ridge-регуляризацией с меньшим alpha

**Гипотеза:** включить `hrv_dfa_alpha1` в interaction-признаки (например, `smo2_x_dfa` вместо `smo2_x_rr`) и проверить влияние на SHAP-ранжирование.

### Признаки с неустановленным механизмом влияния (NIRS std)

**trainred_hhb_std, trainred_thb_std, trainred_hbdiff_std** занимают позиции 2–6 в SHAP для LT2. Физиологических гипотез несколько:
1. СКО отражает дыхательные колебания NIRS (частота дыхания растёт с нагрузкой)
2. СКО отражает кадан-синхронные артефакты педалирования
3. СКО отражает реальную нестабильность оксигенации вблизи порога

Корреляция с `cadence_mean_rpm` — смешанная, паттерн не системный. Гипотеза 2 не подтверждена. **Рекомендация для диссертации:** зафиксировать как открытый вопрос — std-признаки важны по SHAP, но механизм нуждается в верификации.

## Известные ограничения

1. **hrv_mean_rr_ms конфаунд**: ρ_within = +0.997 (сильный сигнал), ρ_between = +0.753 (конфаунд тренированности). Удаление признака увеличивает MAE LT2 на +1.0 мин и катастрофически ухудшает S012. Оставлен с документированным ограничением.

2. **Математические дубликаты в HRV**: hrv_sd1_ms ≡ hrv_rmssd_ms/√2 (r=1.000); hrv_sd2_ms ≈ 1.39×hrv_sdnn_ms (r=0.998). Оба дубликата увеличивают мультиколлинеарность без пользы. Не устранены в v0001–v0006; требуют отдельной версии.

3. **Мультиколлинеарность hrv_sdnn_ms / hrv_sd2_ms** в LT1: Huber и SHAP дают им коэффициенты ±800. Для интерпретации LT1 использовать коэффициенты ElasticNet (v0001) или Ridge (недоступен для LT1).

4. **Малая выборка**: n=13 участников (LT2), n=9 (LT1). LOSO-оценки могут быть нестабильны.

5. **S003 — элитный атлет**: сильный выброс в LT1. HuberRegressor снижает его влияние без удаления из датасета.
