## Этап 4.3. Приведение к научному стилю

### a) Лицо и стиль повествования

- Используется безличная форма либо первое лицо множественного числа: «получено», «было обнаружено», «нами установлено».
- **Не использовать:** «я», «давайте», «как мы видим», «возьмём», «попробуем», «получается так, что».
- Оценочные слова и просторечия убрать: «довольно хорошо», «отличный результат», «к сожалению», «увы», «практически идеально», «к слову». Заменить нейтральными формулировками с числами.

Примеры:
- «Модель показала довольно точные предсказания» → «Прогнозы модели характеризуются MAE = 1,90 мин (95 % ДИ: 1,42–2,38)».
- «Получилось, что HRV-признаки важнее для LT2» → «Признаки ВСР демонстрируют наибольший вклад в предсказание LT2».

### b) Времена глаголов

- Прошедшее (совершенный вид) для выполненных действий: «была обучена», «вычислены», «проведена кросс-валидация».
- Настоящее — для свойств моделей и результатов: «модель демонстрирует», «коэффициент детерминации составляет», «интервал содержит истинное значение в … % случаев».

### c) Числовой формат

- Десятичный разделитель — **запятая**: «1,90», не «1.90». Проверить все числа в тексте.
- Единицы измерения отделять неразрывным пробелом (в Markdown — обычным пробелом, но не переносить через строку, если это в одном предложении): «1,90 мин», «18 испытуемых», «95 %».
- Доверительные интервалы: «1,90 [1,42; 2,38] мин» либо «95 % ДИ: 1,42–2,38 мин».
- p-значения: «p = 0,034» (с пробелами вокруг «=»).
- R² пишется как «R²» (с верхним индексом-юникодом), не «R^2» и не «R2».

### d) Единообразие терминов

Закрепить и не путать:

- «лактатный порог LT1» / «лактатный порог LT2» (не «первый лактатный», не «верхний», не «нижний»).
- «модальность признаков» (не «канал», не «сигнал», когда речь о наборе признаков).
- «LOSO-кросс-валидация» (не «leave-one-out по испытуемым», не «кросс-валидация по испытуемым» в первом упоминании; после первого можно «LOSO»).
- «конформные интервалы» / «эмпирическое покрытие» (не «conformal prediction», не «coverage»).
- «двухэкспертная схема» / «оракул-роутер» / «реальный роутер».
- «целевая переменная» (не «таргет»).
- «признаки» (не «фичи», не «features»).
- «конвейер обработки» (не «pipeline»).
- «опорная модель» (не «baseline»).
- «прогноз» / «предсказание» (синонимично, но в одном абзаце выдерживать одно).

### e) Аббревиатуры

При первом употреблении — полностью с аббревиатурой в скобках:

- «электромиография (ЭМГ)»
- «ближняя инфракрасная спектрометрия (NIRS)»
- «вариабельность сердечного ритма (ВСР, HRV)»
- «средняя абсолютная ошибка (MAE)»
- «коэффициент детерминации (R²)»
- «leave-one-subject-out (LOSO)»

Далее — только аббревиатура. Аббревиатуры, оставляемые латиницей (общепринятые в ML): LOSO, MAE, R², SHAP, NIRS, HRV, EMG/ЭМГ, RBF, TCN, LSTM, GBM, SVR, ElasticNet, Ridge, Huber. Остальное русифицировать.

### f) Заголовки

- Без точек в конце.
- Без эмодзи.
- Без markdown-эмфазы внутри заголовка (`## **3.1 ...**` → `## 3.1 ...`).
- Уровни иерархии:
  - `# 3. Результаты`, `# 4. Заключение по ML-части`
  - `## 3.1 Название раздела`
  - `### 3.1.1 Название подраздела`

### g) Таблицы

- Подпись **над** таблицей: `Таблица 3.X — Название` (тире — короткое или длинное, выбрать один вариант на всю главу).
- Примечания — под таблицей, отдельным абзацем курсивом или с пометкой «*Примечание.* …».
- В шапке столбцов — без markdown-эмфазы.

### h) Рисунки

- Подпись **под** рисунком: `Рисунок 3.X — Название`.
- Все обозначения осей, единицы измерения и пояснения к легенде — в подписи.
- Если рисунок в исходнике обозначен как `![…](путь)` с локальным путём — оставить markdown, но проверить, что подпись отделена от изображения и читается как самостоятельная строка.

### i) Перечни

- Маркированные списки оставлять, когда есть параллельная структура из 4+ пунктов.
- Перечни из 2–3 коротких пунктов — переписать связным предложением.

### j) Английские вкрапления

Перевести:
- «pipeline» → «конвейер»
- «baseline» → «опорная модель» / «базовая модель»
- «feature» → «признак»
- «target» → «целевая переменная»
- «predict» / «prediction» → «прогноз» / «предсказание»
- «threshold» → «порог»
- «window» → «окно»
- «outlier» → «выброс»
- «cross-validation» → «кросс-валидация» (написание через дефис)

### k) Цитирование числовых утверждений

Каждое числовое утверждение должно сопровождаться:
- либо ссылкой на таблицу/рисунок данной главы («…(таблица 3.4)»);
- либо явным указанием в предложении («полученное значение MAE = 1,90 мин»).

Если число встречается без подтверждения и в таблицах его нет — пометить в журнале правок и обратиться к пользователю.

### l) Запятые, кавычки, тире

- Кавычки русские «ёлочки» для прямой речи и устойчивых выражений; внутренние кавычки — „лапки“.
- Тире длинное «—» для пояснений, дефис «-» — только в составных словах.
- В диапазонах чисел использовать тире без пробелов или короткое тире: «1,42–2,38 мин».

## Этап 4.4. Финальная сквозная вычитка

После применения правил 4.3 — прочитать `chapter_3_results_FULL.md` подряд от начала до конца. На этом проходе проверить:

1. **Согласованность чисел.** Где встречается главная цифра — везде ли она одинакова?
   - 1,90 мин (не 1,86 / 1,9 / 1.90).
   - Победители inner-моделей соответствуют опорным фактам из плана.
2. **Параметры моделей.** Приводятся **ровно один раз** — при первом упоминании конкретной модели. Дальше — без параметров.
3. **Сквозные ссылки.** Все «см. раздел 3.X» ведут на существующие разделы; все «таблица 3.X» и «рисунок 3.X» — на существующие объекты.
4. **Целостность повествования.** Нет «оборванных хвостов» от удалённых ссылок и определений. Нет фраз вида «как показано выше», если выше ничего не показано.
5. **Единообразие терминов.** Один термин — одно написание во всём документе.
6. **Расшифровки аббревиатур.** Первое упоминание — полное; далее — аббревиатура.
7. **Логика заголовков.** Нумерация без пропусков, иерархия не нарушена.

При обнаружении проблем — править сразу. Если правка требует решения, выходящего за инструкции (например, противоречие между двумя разделами, которое не разрешается стилистически) — выписать в журнал и оставить помеченным `<!-- TODO для пользователя: ... -->`.
