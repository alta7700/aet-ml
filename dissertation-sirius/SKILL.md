## Ключевые правила оформления (всегда в памяти)

### Параметры страницы
- Поля: левое **30 мм**, правое **15 мм**, верхнее/нижнее **20 мм**
- Шрифт: **Times New Roman, 14 пт**, чёрный
- Интервал: **1,5** строки в тексте; **1,0** в таблицах и подписях рисунков
- Абзацный отступ: **1,25 см** единообразно по всему тексту
- Выравнивание: **по ширине**
- Нумерация страниц: арабские цифры, **по центру нижнего поля**, без точки
- Первый лист (титул): номер **не проставляется**
- Каждый раздел (глава) начинается с **новой страницы**

### Объём МД
**70–100 страниц** без приложений. Ориентир по разделам:
- Введение: 2–3 стр.
- Обзор литературы: 20–30 стр.
- Материалы и методы: 10–20 стр.
- Результаты и обсуждение: 25–35 стр.
- Заключение: 3–5 стр.
- Список источников: 3–7 стр.

### Заголовки
- Уровень 1 (разделы/главы): **ПРОПИСНЫЕ БУКВЫ**, прямой полужирный
- Уровень 2 (подразделы): строчные, прямой полужирный
- Уровень 3 (пункты): строчные, полужирный курсив
- Точка в конце номеров и заголовков **не ставится**
- Перенос слов в заголовках **запрещён**

### Нумерация объектов
- Таблицы, рисунки, формулы — **в пределах раздела**: «Рисунок 3.2» = 2-й рисунок 3-го раздела
- На каждый объект — ссылка в тексте **до его появления**
- Заимствованный объект → ссылка `[Автор, год]` в тексте при упоминании

### Биологические объекты
- Гены — *курсив*, именование по HGNC
- Латинские виды и термины (*in silico*, *et al.*) — *курсив*
- При **первом** появлении: название + идентификатор в базе (RefSeq / Ensembl / UniProt / KEGG / GO)

### Ссылки на источники (стиль шаблона)
- 1–3 автора: `[Katoh, Standley, 2013]`
- >3 авторов: `[Li и др., 2010]`
- Несколько подряд: `[Akberdin и др., 2018; Kulyashov и др., 2023]`
- Несколько работ одного года: суффиксы `2024a`, `2024b`
- Нумерация в списке — **по порядку первого упоминания** в тексте (п. 4.10.4)

### Реферат
- Объём: **≤ 850 знаков без пробелов**, не более 1 страницы А4
- Ключевые слова: **5–15**, именительный падеж, **ПРОПИСНЫЕ**, через запятую
- Пишется **последним** — после финальной вёрстки, когда известны точные числа

### Сокращения
- Отдельный раздел — только если аббревиатур **> 3** (п. 4.5.2)
- Порядок: алфавитный, сначала кириллица, потом латиница
- В оглавление **не включается** (п. 4.6.3)
- Двустороннее соответствие: каждая аббревиатура из раздела встречается в тексте, и наоборот

## Как работать с этим скиллом

### Если пользователь просит написать раздел
1. Прочитай `references/dissertation_map.md` — что именно должно быть в этом разделе
2. Прочитай `references/entities_dependencies.md` — какие сущности он наследует от других разделов
3. Напиши текст, строго следуя требованиям стандарта
4. Укажи, какие данные пользователь должен подставить сам (цель, задачи, конкретные результаты)

### Если пользователь просит проверить текст
1. Прочитай `references/entities_dependencies.md`, раздел «Чеклист согласованности»
2. Пройди по каждому пункту применительно к предоставленному тексту
3. Отметь найденные рассинхроны и дай конкретные формулировки для исправления

### Если пользователь работает с литобзором
1. Прочитай `references/litreview_plan.md` — актуальный план обзора со всеми подразделами
2. Для поиска статей — промпты для Perplexity в `references/domain_knowledge.md` раздел 12
3. Для систематизации PDF — промпт для Claude там же (отправлять с прикреплёнными PDF)
4. Gap центральный: раздел 6.4 плана — отсутствие мультимодальных онлайн-систем (ЭМГ + NIRS + HRV) для предсказания двух порогов одновременно


Направь к `references/assembly_plan.md`. Ключевое: порядок написания **почти обратен** порядку в PDF.

### Если пользователь просит объяснить gap
Gap — это явная формулировка того, чего не хватает в существующей литературе. Он должен появиться в Обзоре и затем стать обоснованием новизны в Заключении. Без gap'а новизна «повисает в воздухе».

### Если пользователь запускает новую версию модели (v000X)
1. Сохрани артефакты: предсказания по субъектам, MAE / MAE_pre / MAE_post / MAE_near, параметры модели
2. Интерпретируй результаты сразу — что улучшилось, где модель ошибается, какие субъекты проблемные
3. Добавь версию в пересчёт по фазам (pre / near / post) — не откладывай на потом

### Правила ведения `references/` файлов
- Только факты в настоящем времени — без истории, без «мы решили», без «раньше было»
- Не упоминать то, чего нет в реализации (убрали RF, Moxy, push/pull — в файлах их нет)
- Не писать «рекомендуется», «возможно», «если нужно» — либо факт, либо `<!-- TODO -->`
- Зафиксированная терминология: **load/rest** (не push/pull), **Train.Red** (не Moxy), **VL дист + VL прокс** (не RF), таргет — регрессия **time_to_lt_sec**
- Работать в локальной папке `dissertation-sirius/references/`, затем синхронизировать со skills-plugin

## Запрещённые ошибки (предупреждай пользователя)

- Реферат написан до финальной вёрстки → числа будут неверными
- Введение написано до Обзора → актуальность не опирается на gap
- Задач в ТЗ больше, чем выводов в Заключении
- Источники пронумерованы по алфавиту, а не по первому упоминанию
- Тема на титуле не совпадает с формулировкой в приказе
- В Обсуждении цитируется работа, которой нет в Обзоре
- Раздел «Сокращения» включён в Оглавление
- Рисунок или таблица появляются в тексте без предшествующей ссылки
- Приложение включено без ссылки на него в основном тексте
