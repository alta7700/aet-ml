## Оформление (Сириус 01-2023)

- Поля: лево **30**, право **15**, верх/низ **20** мм
- Шрифт: **Times New Roman 14**, интервал **1,5** (в таблицах/подписях — 1,0), отступ **1,25 см**
- Заголовки ур.1 — **ПРОПИСНЫЕ**, ур.2 — полужирный, ур.3 — полужирный курсив; точка не ставится
- Нумерация объектов в пределах раздела: «Рисунок 3.2»; ссылка в тексте — до объекта
- Ссылки: `[Katoh, Standley, 2013]` / `[Li и др., 2010]`; нумерация в списке — по первому упоминанию
- Реферат: ≤ 850 знаков б/п, 5–15 ключевых слов ПРОПИСНЫМИ; писать последним
- Раздел «Сокращения»: только если аббревиатур > 3; в оглавление не включается
- Объём: **70–100 стр.** без приложений

## Правила работы

**Написание раздела:** читай `dissertation_map.md` + `entities_dependencies.md`, затем пиши.

**Проверка текста:** читай чеклист в `entities_dependencies.md`, отмечай рассинхроны.

**Gap:** отсутствие мультимодальных систем (ЭМГ + NIRS + HRV) для предсказания двух порогов одновременно. Появляется в Обзоре → обосновывает новизну в Заключении.

**Порядок написания:** почти обратен порядку в PDF → `assembly_plan.md`.

**Новая версия модели (v000X):**
1. Сохрани артефакты: предсказания, MAE / MAE_pre / MAE_post / MAE_near
2. Интерпретируй сразу — что улучшилось, где ошибается, какие субъекты проблемные
3. Сразу добавь в пересчёт по фазам (pre / near / post)

**`references/` файлы — только факты:**
- Настоящее время, никакой истории
- Не упоминать то, чего нет: нет RF, нет Moxy, нет push/pull
- Вместо «возможно» / «рекомендуется» — либо факт, либо `<!-- TODO -->`
- Терминология: **load/rest**, **Train.Red**, **VL дист + VL прокс**, таргет — **time_to_lt_sec**

## Запрещённые ошибки

- Реферат до финальной вёрстки
- Введение до Обзора
- Задач больше, чем выводов в Заключении
- Источники по алфавиту, а не по первому упоминанию
- Тема на титуле ≠ формулировка в приказе
- Цитата в Обсуждении без упоминания в Обзоре
- «Сокращения» в оглавлении
- Объект без предшествующей ссылки в тексте
