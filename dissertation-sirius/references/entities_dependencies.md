# Сквозные сущности: зависимости между разделами

> Источник: Стандарт АНО ВО «Университет «Сириус» 01-2023 + Шаблон «Системная биология»  
> Назначение: контроль согласованности — если сущность изменилась, правятся все связанные места

---

## Принцип

Сквозная сущность — это объект (формулировка, число, термин, список), который **появляется в нескольких разделах** и должен быть там одинаковым или логически согласованным.  
Рассинхрон сущностей — самая частая причина замечаний на защите.

---

## Главная цепочка зависимостей

```
Задача → Метод → Результат → Вывод
```

Каждое звено обязано существовать. Обрыв в любой точке — структурная ошибка:
- есть задача без результата → работа не выполнена
- есть вывод без задачи → вывод ни к чему не привязан
- есть метод, который нигде не применяется → раздел лишний

## Таблица сущностей

### 1. Тема

| Атрибут | Значение |
|---|---|
| Где фигурирует | Приказ → Титул → ТЗ → (подразумевается везде) |
| Требование | Дословное совпадение во всех трёх местах |
| Когда меняется | Только через переутверждение в деканате/дирекции |
| Риск рассинхрона | Правка формулировки «для благозвучия» на титуле без правки приказа |

---

### 2. Цель

| Атрибут | Значение |
|---|---|
| Где фигурирует | ТЗ → Реферат (рус.) → Abstract → Введение → Заключение (оценка достижения) |
| Требование | Одна формулировка во всех пяти местах |
| Типичная ошибка | В ТЗ — «разработать модель», в Заключении — «изучить механизм» |
| Проверка | grep по ключевым словам цели во всех пяти разделах |

---

### 3. Задачи

| Атрибут | Значение |
|---|---|
| Где фигурируют | ТЗ (нумерованный список) → Введение → структура глав Результатов → Заключение (вывод по каждой) |
| Требование | Каждая задача должна иметь: (а) блок в Материалах/Методах, (б) блок в Результатах, (в) вывод в Заключении |
| Количество выводов | ≥ числа задач |
| Типичная ошибка | 4 задачи в ТЗ, 3 вывода в Заключении |

**Матрица покрытия задач** (заполнять при написании):

| № задачи | Формулировка | Раздел м/м | Раздел результатов | Вывод в заключении |
|---|---|---|---|---|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |
| 4 | | | | |

---

### 4. Гипотеза

| Атрибут | Значение |
|---|---|
| Где фигурирует | Введение (формулировка) → Обзор (обоснование через gap) → Результаты (проверка) → Заключение (подтверждение / опровержение) |
| Если не сформулирована явно | Задачи выполняют её роль как проверяемый каркас |
| Требование | Результаты обязаны адресовать гипотезу, даже если она опровергнута |

---

### 5. Объект и предмет

| Атрибут | Значение |
|---|---|
| Где фигурируют | Реферат → Введение |
| Требование | Одна формулировка в обоих местах; объект — шире предмета |
| Типичная ошибка | В реферате «метаболическая сеть», во введении «геном организма» |

---

### 6. Методы

| Атрибут | Значение |
|---|---|
| Где фигурируют | Реферат (перечень) → Введение (упоминание) → Обзор (обоснование выбора) → Материалы и методы (детали) → Результаты (применение) |
| Правило полноты | Любой метод из реферата → описан в м/м → применён в результатах |
| Правило чистоты | Не должно быть метода в м/м, который нигде не применяется |
| Обоснование | Выбор метода обосновывается в Обзоре (через сравнение с альтернативами) |

---

### 7. Ключевые слова

| Атрибут | Значение |
|---|---|
| Где фигурируют | Реферат рус. (строчка после объёма) → Abstract (keywords) |
| Количество | 5–15 слов/словосочетаний |
| Форма | Именительный падеж, **ПРОПИСНЫЕ БУКВЫ**, через запятую |
| Требование | Каждое ключевое слово встречается в основном тексте как содержательный термин |
| Перевод в Abstract | Не дословный, а терминологически корректный английский |

---

### 8. Термины и сокращения

| Атрибут | Значение |
|---|---|
| Где фигурируют | Раздел «Сокращения» + первое упоминание в тексте |
| Порог включения | >3 нестандартных сокращений → обязателен отдельный раздел (п. 4.5.2) |
| Двустороннее соответствие | Всё в списке → встречается в тексте; всё нестандартное в тексте → есть в списке |
| Исключения | Стандартные (ЭВМ, вуз, т.е., НИИ) — не расшифровываются |
| Порядок | Алфавитный: сначала кириллица, потом латиница |
| В тексте | При первом упоминании: полное название (сокращение) |

---

### 9. Источники

| Атрибут | Значение |
|---|---|
| Где фигурируют | Ссылки в тексте [N] или [Автор, год] → Список в конце |
| Нумерация | По порядку **первого упоминания** в тексте (п. 4.10.4) |
| Правило чистоты | Каждый источник из списка — ≥1 ссылки в тексте; источников без ссылок быть не должно |
| Формат | ГОСТ Р 7.0.100 + ГОСТ 7.80; допускаются сокращения по ГОСТ 7.11, ГОСТ Р 7.0.12 |

**Стиль ссылок по шаблону системной биологии:**
- 1–3 автора: `[Katoh, Standley, 2013]`
- >3 авторов: `[Li и др., 2010]`
- Несколько источников подряд: `[Akberdin и др., 2018; Kulyashov и др., 2023]`
- Суффиксы при нескольких работах одного года: `2024a`, `2024b`

### 10. Числовые показатели объёма (реферат)

| Атрибут | Значение |
|---|---|
| Где фигурируют | Первая строка Реферата |
| Состав | страниц / рисунков / таблиц / источников / приложений |
| Когда считать | **После финальной вёрстки** — поэтому реферат пишется последним |
| Типичная ошибка | Писать реферат до финальной пагинации, потом не обновить числа |

---

### 11. Результаты

| Атрибут | Значение |
|---|---|
| Где фигурируют | Реферат (сжато, 1–2 предложения) → Результаты гл. 3 (подробно) → Заключение (резюме) |
| Требование | Одна интерпретация во всех трёх местах; не должно быть результата в гл. 3, который не попал в Заключение |

---

### 12. Научная новизна

| Атрибут | Значение |
|---|---|
| Где фигурирует | Реферат (заявлена) → Введение (обоснована ссылкой на gap в Обзоре) → Заключение (подтверждена конкретными результатами) |
| Цепочка валидности | Новизна вытекает из gap'а → gap выявлен в Обзоре → gap закрыт результатами |
| Типичная ошибка | Новизна заявлена, но в Обзоре не показано, чего именно не хватало в литературе |

---

### 13. Практическая значимость

| Атрибут | Значение |
|---|---|
| Где фигурирует | Реферат → Введение → Заключение |
| Требование | Должна быть подкреплена конкретным результатом, а не только применённой методикой |

| Атрибут | Значение |

## Карта зависимостей (граф)

```
                    ┌──────────────────────┐
                    │     ТЗ (контракт)    │
                    │  Тема / Цель / Задачи│
                    └──────────┬───────────┘
                               │ копируется дословно
           ┌───────────────────┼──────────────────────┐
           ▼                   ▼                      ▼
      Введение           Реферат (рус.)          Abstract
    (актуальность +    (объём + кл.слова +    (то же по-англ.)
     те же цель/задачи)  те же результаты)
           │
           │ обоснование gap'а
           ▼
       Обзор лит.
    (gap → новизна)
           │ выбор метода
           ▼
     Материалы и методы
    (биообъекты, термины,
     первые сокращения)
           │ применение метода
           ▼
    Результаты (гл. 3)
    (закрывает каждую задачу,
     таблицы/рисунки с ссылками)
           │ резюме
           ▼
       Заключение
    (вывод по каждой задаче,
     новизна, значимость)
           │
           ▼
    ┌──────────────────┐
    │ Список источников│  ← нумерация по порядку первого упоминания
    └──────────────────┘
    ┌──────────────────┐
    │   Сокращения     │  ← двустороннее соответствие с текстом
    └──────────────────┘
    ┌──────────────────┐
    │   Приложения     │  ← только если есть ссылка в тексте
    └──────────────────┘
```

## Чеклист согласованности (прогонять перед сдачей)

- [ ] Тема совпадает дословно: приказ = титул = ТЗ
- [ ] Цель одинакова: ТЗ = реферат = abstract = введение = заключение
- [ ] Каждая задача закрыта результатом и выводом
- [ ] Каждый метод из реферата описан в м/м и применён в результатах
- [ ] Каждый источник из списка имеет ≥1 ссылки в тексте
- [ ] Нет ссылок в тексте на источники, которых нет в списке
- [ ] Каждая аббревиатура в разделе «Сокращения» встречается в тексте
- [ ] Каждая нестандартная аббревиатура в тексте есть в разделе «Сокращения»
- [ ] На каждую таблицу и рисунок есть ссылка в тексте до его появления
- [ ] На каждое приложение есть ссылка в тексте
- [ ] Все приложения перечислены в оглавлении
- [ ] Числа в реферате (стр./рис./табл./ист./прил.) соответствуют финальной вёрстке
- [ ] Гены — курсивом; латинские названия видов — курсивом
- [ ] При первом появлении биообъектов указаны идентификаторы в базах
- [ ] Gap в обзоре явно обосновывает новизну из заключения
