# Структура ML-части диссертации — навигатор

## Ответ на вопрос «достаточно ли 2.8.1–2.8.4?»

Нет. Структура «постановка / baseline / расширенная модель / валидация» описывает
работу с 2 моделями. У нас фактически: зоопарк из 10 версий (1 линейная + 9 NN),
5 модальностей, ablation абсолютных признаков, гипотеза «два эксперта», conformal,
SHAP. Нужна более широкая разбивка — ниже.

## Где ML-часть в финальном PDF

- **Глава 2 «Материалы и методы»** — подразделы 2.8 (модели) и 2.9 (статобработка).
  Файлы 01–06.
- **Глава 3 «Результаты и обсуждение»** — файлы 07–11.
- **Глава 4 «Заключение»** — файл 12 (ML-вклад в общие выводы).

Нумерация пунктов (2.8.1, 3.x.x) — ориентир, подгонишь под общую структуру работы.

## Порядок написания (важно!)

По стандарту «Сириуса» пишется **снизу вверх**:
1. Сначала **методы** (01–06) — пока детали свежи.
2. Потом **результаты черновиком** (07–10) — чистые факты, без обсуждения.
3. Потом **обсуждение** (11) — интерпретация, сравнение с литобзором.
4. Заключение (12) — по задачам ТЗ как по чеклисту.
5. Введение и реферат — в самом конце (не в этой папке).

Каждый метод из 01–06 обязан быть **применён** в результатах 07–10.
Каждый результат обязан **закрывать задачу из ТЗ**.

## Источник всех чисел

Папка `results/final/` — туда сложены все артефакты по шагам:
- `dissertation_tables/` — 5 готовых CSV-таблиц для прямого переноса;
- `README.md` — 7 кратких выводов + навигация;
- план работ и отчёты по шагам — в `final-plan/`.

Не пересчитывай числа вручную — бери из `dissertation_tables/`.
