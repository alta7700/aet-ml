# 2.8.1 Постановка задачи машинного обучения

## О чём писать
- Тип задачи: **регрессия** — предсказание времени (в секундах) до порога
  по текущему скользящему окну сигналов.
- Два независимых таргета: `time_to_LT1` (аэробный порог) и `time_to_LT2`
  (анаэробный порог). Подчеркнуть: это две **разные** подзадачи, не одна.
- Единица предсказания — каузальное скользящее окно (признаки только из прошлого,
  без заглядывания вперёд). Сослаться на раздел про нарезку окон.
- Калибровочная ступень 30 Вт исключена из обучающих окон (используется только
  для базальных признаков — см. 2.8.2).
- Целевая переменная — оставшееся время до порога; на инференсе монотонно
  убывает по ходу теста.

## Откуда брать
- Определение окон и таргетов — из своих разделов про датасет (`DATASET_PIPELINE.md`).
- Числа: 4130 окон для LT1, 3651 для LT2 (после фильтрации), 18 испытуемых.

## Как собирать
1–1,5 страницы. Сначала словами «что предсказываем и зачем», потом формальная
постановка (X — вектор признаков окна, y — секунды до порога). Обязательно
оговорить каузальность — это методологически важно и спросят на защите.

## Связи
- Питает: все последующие подразделы.
- Закрывает задачу ТЗ: формулировку «построить модель предсказания порога».
