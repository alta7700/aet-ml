# 2.8.2 Признаковое пространство и модальности

## О чём писать
- Три модальности признаков и их состав:
  - **EMG** — z-нормированные признаки ЭМГ + кинематика педалирования (~79 призн.);
  - **NIRS** — Train.Red SmO₂/HHb/HbDiff/THb + running-признаки (~18);
  - **HRV** — 7 признаков из RR: mean_RR, SDNN, RMSSD, SD1/SD2, SD1/SD2-ratio, DFA-α1.
- Комбинации (feature sets): EMG, NIRS, HRV, EMG+NIRS, EMG+NIRS+HRV.
  Зачем тестировались все — чтобы определить минимально достаточный набор сенсоров.
- Вариант **with_abs / noabs**: noabs убирает 6 «абсолютных» признаков
  (`hrv_mean_rr_ms`, абсолютные средние NIRS, interaction). Гипотеза, которую это
  проверяет: достаточно ли одних трендов без индивидуальных «норм».
- Базальные признаки на 30 Вт (`hrv_hr_baseline_bpm`, `hrv_rmssd_baseline_ms`,
  EMG/SmO₂ baseline) — отдельный набор для маршрутизатора (см. 2.8.3 / результат 09).

## Откуда брать
- Состав признаков — `dataset_pipeline/` + skill-файлы препроцессинга.
- EXCLUDE_ABS (что убирает noabs) — `scripts/v0011_modality_ablation.py`.

## Как собирать
1,5–2 страницы. Таблица «модальность → группы признаков → количество».
Чётко развести: feature set (что подаём) vs variant (with/without absolute).
Это понадобится во ВСЕХ результатах — разметь термины аккуратно один раз.
