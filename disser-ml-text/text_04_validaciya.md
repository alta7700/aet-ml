# 2.8.4 Схема валидации

## Выбор стратегии валидации

В качестве единственной схемы валидации во всех экспериментах применялась
стратегия Leave-One-Subject-Out (LOSO). Суть метода: на каждой итерации один
испытуемый целиком выносится в тестовое множество, остальные 17 формируют
обучающее множество; процедура повторяется для каждого из 18 испытуемых.

Выбор LOSO обусловлен целевой областью применения модели: предсказание должно
работать на **новом, ранее не встречавшемся человеке**, а не только на субъектах,
чьи данные присутствовали при обучении. Альтернативные стратегии (случайное
разбиение окон, k-fold по окнам) приводят к утечке субъект-специфичной информации
между обучающим и тестовым множеством, искусственно завышают метрики и не
позволяют оценить способность модели к кросс-субъектному обобщению. При n = 18
испытуемых LOSO к тому же максимизирует объём обучающего множества на каждой
итерации.

## Объём выборки и исключения

Полная схема LOSO включала **n = 18 испытуемых** для всех конфигураций линейного
семейства (v0011) и для всех таргетов в нейросетевых версиях при предсказании LT1.
Часть нейросетевых архитектур при предсказании LT2 охватывает 16–17 испытуемых:
версии v0102 и v0105 — **n = 16**, версии v0103, v0104, v0106a, v0106b, v0106c и
v0107 — **n = 17**. Расхождение обусловлено внутренними операциями удаления
пропущенных значений в соответствующих скриптах обучения, которые в ряде
конфигураций исключали данные одного-двух испытуемых.
При сравнении моделей с разным n парные статистические тесты применяются только
на пересечении субъектов (раздел 2.9).

Малая выборка (n = 18) является ограничением настоящей работы; его последствия
обсуждаются в разделе [ВСТАВИТЬ: ссылка на раздел «Ограничения»].

## Предотвращение утечки данных

На каждой итерации LOSO предобработка признаков обучалась **исключительно на
обучающем множестве** и затем без переобучения применялась к тестовому. В частности:

- **Импутация пропущенных значений** (замена медианой) — медианы по признакам
  вычислялись на обучающем множестве фолда и применялись к тестовому субъекту.
- **Стандартизация признаков** (z-score) — среднее и стандартное отклонение
  вычислялись на обучающем множестве; тестовые значения пересчитывались с
  этими же параметрами.
- **Per-subject z-нормировка ЭМГ** (раздел 2.8.2) проводилась на уровне
  отдельного испытуемого по его собственным данным первой ступени нагрузки и
  не предполагала обращения к данным других субъектов; утечки не возникает.

Таким образом, все нормировочные статистики, применявшиеся к тестовому субъекту,
были получены из обучающего множества и не содержат информации о тестовом
субъекте.

## Пересборка per-subject метрик из npy-артефактов

Предсказания всех моделей сохранялись в бинарные файлы в формате отсортированных
по субъекту последовательностей окон. На этапе финальной агрегации метрик
выполнялась детерминированная пересборка: разбиение окон по субъектам
восстанавливалось по тем же правилам фильтрации и сортировки, что применялись
при обучении, и для каждого субъекта вычислялась индивидуальная MAE по его
тестовым окнам.

Полученные per-subject значения MAE сверялись с ранее записанными сводными
файлами: расхождение для нейросетевых версий составило не более 0,01 мин
(максимальное |Δ|), что подтверждает детерминизм процедуры и консистентность
артефактов. Детерминизм линейных моделей гарантирован отсутствием стохастических
операций в sklearn-зоопарке (раздел 2.8.3). Для нейросетевых версий
воспроизведение предполагает использование сохранённых файлов предсказаний,
а не переобучение.

Итоговое покрытие после пересборки: для LT1 — все 10 архитектурных версий на
18 субъектах; для LT2 — линейная v0011 на 18 субъектах, нейросетевые — на 16–17
субъектах (по версиям, см. выше).
