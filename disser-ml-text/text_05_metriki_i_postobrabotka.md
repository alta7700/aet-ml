# 2.8.5 Метрики качества и постобработка предсказаний

## Метрики качества

### Три варианта MAE

Центральной метрикой в работе служит средняя абсолютная ошибка (MAE, mean
absolute error). Поскольку при LOSO-валидации на 18 субъектах отдельные субъекты
вносят в итоговую статистику неодинаковое число окон, различаются три связанных,
но различных показателя:

1. **Субъект-взвешенная медиана MAE** (в таблицах: *loso_mae_median*) — для
   каждого из 18 испытуемых вычислялась MAE по его тестовым окнам; итоговый
   показатель — медиана этих 18 значений. Каждый испытуемый имеет вес 1
   вне зависимости от числа его окон. Это **основная метрика** в сводных
   таблицах настоящей работы.

2. **Медиана по окнам** — медиана абсолютных ошибок по всем тестовым окнам
   объединённо (каждое окно имеет вес 1). Устойчива к выбросам; применяется
   при анализе распределения ошибки.

3. **Среднее MAE по окнам** (в таблицах: *raw MAE*) — среднеарифметическое
   абсолютных ошибок по всем тестовым окнам всех субъектов. Субъекты с
   большим числом окон имеют больший вес.

Соотношение между показателями: субъект-взвешенная медиана систематически ниже
среднего по окнам при наличии испытуемых с высоким разбросом ошибок: тяжёлые
случаи порождают длинный правый хвост при усреднении по окнам, но учитываются
как одна точка при субъект-взвешенной медиане. При цитировании метрик в тексте
всегда указывается, какой именно из трёх показателей приводится.

Единица измерения для всех трёх вариантов — **минуты** (целевая переменная
конвертируется из секунд при подсчёте MAE делением на 60).

### Коэффициент детерминации R²

Вычислялся стандартным образом по формуле
$$
R^2 = 1 - \frac{\sum_i (y_i - \hat{y}_i)^2}{\sum_i (y_i - \bar{y})^2},
$$
где суммирование ведётся по всем тестовым окнам всех испытуемых совместно (взвешивание по окнам, аналогично raw MAE). В рамках LOSO $R^2 < 0$ для конкретного фолда означает, что модель не извлекла никакой полезной информации из физиологических сигналов и предсказывает хуже, чем константа $\bar{y}$, вычисленная по обучающему множеству. Доля субъектов с $R^2 < 0$ отслеживалась отдельно как индикатор робастности модели.

### Ранговая корреляция Спирмена (ρ)

Коэффициент ранговой корреляции Спирмена вычислялся между предсказанными и
истинными значениями по всем тестовым окнам. Этот показатель устойчив к
монотонным нелинейным искажениям шкалы и особенно информативен для задачи
«когда наступит порог»: высокий ρ означает, что модель правильно ранжирует
окна по близости к порогу даже при систематическом смещении абсолютных
значений.

## Честные базовые предсказатели

Для оценки нижней границы сложности задачи использовались два тривиальных
предсказателя:

- **MeanPredictor** — предсказывает константу, равную среднему значению
  целевой переменной по обучающему множеству фолда; не использует ни
  физиологические сигналы, ни временну́ю структуру теста. Служит абсолютной
  нижней границей: модель, не превосходящая MeanPredictor, не извлекает
  никакой полезной информации.
- **FirstWindowPredictor** — предсказывает для каждого окна значение целевой
  переменной, зафиксированное в первом окне сессии данного испытуемого.
  Поскольку время до порога монотонно убывает в ходе теста, первое окно несёт
  информацию о начальной удалённости от порога даже без обращения к сигналам.
  Этот предсказатель проверяет более строгое условие: превзойти его означает,
  что физиологические сигналы добавляют информацию сверх одной лишь временно́й
  структуры нагрузочного теста.

Обе базовые линии воспроизводились в рамках LOSO совместно с основным
экспериментом. Превышение точности модели над соответствующей базовой линией
количественно характеризует вклад физиологических сигналов в предсказание.

## Фильтр Калмана как постобработка

Предсказания модели по отдельным окнам содержат высокочастотные флуктуации,
не согласующиеся с физиологическим ожиданием монотонного убывания времени до
порога. Для сглаживания последовательности предсказаний применялся
**одномерный фильтр Калмана** с физически мотивированной моделью перехода:

$$
x_t = x_{t-1} - \Delta t + w_t, \quad w_t \sim \mathcal{N}(0,\, \sigma_p^2),
$$
$$
z_t = x_t + v_t, \quad v_t \sim \mathcal{N}(0,\, \sigma_{\mathrm{obs}}^2),
$$

где $x_t$ — состояние (оценка оставшегося времени в секундах), $\Delta t = 5{,}0$
с — шаг окна, $z_t$ — предсказание «сырой» модели для окна $t$,
$\sigma_p = 15{,}0$ с — шум модели перехода (фиксирован),
$\sigma_{\mathrm{obs}}$ — шум наблюдения (подбирался по сетке).

Фильтрация проводилась раздельно для каждого испытуемого в пределах его тестового
фолда. Значение $\sigma_{\mathrm{obs}}$ перебиралось по сетке
$\{30, 50, 75, 150\}$ с; для каждой конфигурации (версия × набор признаков × таргет) выбиралось
$\sigma_{\mathrm{obs}}$, минимизирующее MAE после фильтрации на полном тестовом
множестве. Эффект сглаживания количественно описывается показателем

$$
\Delta_{\mathrm{Kalman}} = \mathrm{MAE}_{\mathrm{raw}} - \mathrm{MAE}_{\mathrm{Kalman}},
$$

положительные значения которого соответствуют улучшению при применении фильтра,
отрицательные — ухудшению (избыточное сглаживание).

Фильтр Калмана не является компонентом процедуры обучения и не нарушает
каузальности: на момент обновления оценки для окна $t$ используются только
предсказания модели для окон $1, \ldots, t$.
