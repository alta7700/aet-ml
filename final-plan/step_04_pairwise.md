# Шаг 03. Парные сравнения моделей-кандидатов

## Цель
Получить право писать в тексте «модель X статистически значимо лучше Y», а не «MAE на 0.1 мин ниже».

## Входы
- Список кандидатов из `results/final/05_candidates_intersection.csv` (шаг 01)
- `results/<version>/per_subject.csv` всех версий из списка

## Что считаем
Для фиксированного `(target, feature_set, variant)`:
1. Берём список моделей-кандидатов в этой группе.
2. Для каждой пары моделей считаем **paired Wilcoxon** на per-subject MAE (только субъекты, присутствующие в обоих файлах).
3. Применяем **поправку Holm** на множественное сравнение внутри группы.
4. Сохраняем: модели A/B, n_paired, медиана разницы, p, p_holm, вердикт.

## Артефакты
- `scripts/final_pairwise_wilcoxon.py`
- `results/final/07_pairwise_wilcoxon.csv`
- `results/final/plots/pairwise_heatmap_<target>_<fset>.png` — матрица p_holm для каждой группы кандидатов
- Подпись в `results/final/README.md`

## Ограничения
- `n_paired` обязан быть ≥12, иначе строку помечаем `underpowered=True` и в текст не выносим.
- Только two-sided тест (мы не знаем заранее, кто лучше).

## Критерий завершённости
- Для каждой целевой группы есть либо победитель (p_holm<0.05), либо явная пометка «различия незначимы».
- В тексте диссертации можно ссылаться на конкретные строки CSV.
