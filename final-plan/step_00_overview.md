# Общий план превращения `results/` в материал диссертации

## Контекст
- Все модели обучены LOSO на n=18.
- В `results/summary_all_versions.csv` лежат агрегированные метрики 9 версий (v0011 + v0102…v0107, v0101).
- Колонки: `raw_mae_min`, `kalman_mae_min`, `kalman_{30,50,75,150}`, `r2`, `rho`, `n_windows`, `best_sigma_obs`.
- В `results/<version>/per_subject.csv` лежат LOSO-результаты по субъектам.

## Известные проблемы данных
- **v0011** имеет другую схему `per_subject.csv`: вместо `variant` — колонка `model` (зоопарк sklearn).
- **per_subject NN-моделей неполный**: v0101 — 18 субъектов, v0103–v0107 — 17, v0102/v0105 — 16. Парные сравнения только на пересечении субъектов.
- **shap_conformal/** старый, не доверять, пересчитывать.
- Для v0011 ablation `with_abs vs noabs` на EMG вырождена (значения совпадают до знака — EMG абсолютных признаков не имеет в данной реализации).

## Главные метрики
- **Основная**: `loso_mae_median` (медиана MAE по субъектам LOSO).
- **Со-главная**: `rho` (Spearman) — ранговое качество.
- `raw_mae_min` оставляем как техническую (взвешена по числу окон субъекта).

## Прокси «переобученности»
1. `loso_mae_std` — разброс по субъектам;
2. `gap = loso_mae_mean − raw_mae_min` — оптимизм агрегата;
3. `loso_neg_r2_share` — доля субъектов с R²<0.

## Шаги
- **step_01** — расширение `session_params.parquet`: добавить `hrv_hr_baseline_bpm` и `hrv_rmssd_baseline_ms` на калибровке 30 Вт (нужно для шага 05).
- **step_02** — единый ranker + Top-K + пересечения (3 уровня группировки).
- **step_02a** — распределение per-subject ошибок у топ-моделей (фундамент шага 05).
- **step_02b** — анализ ошибки по времени до порога (time-resolved).
- **step_03** — ablation `with_abs vs noabs`.
- **step_04** — парные сравнения кандидатов (Wilcoxon с Holm-поправкой).
- **step_05** — гипотеза «два эксперта» (trained/untrained), включая маршрутизатор.
- **step_06** — пересчёт SHAP и conformal на финальных моделях.
- **step_07** — финальная папка `results/final/` для диссертации (таблицы, графики, подписи).

## Артефакты
Каждый шаг кладёт результаты в `results/final/` с пометками в `README.md` той же папки. Скрипты — в `scripts/` с префиксом `final_`.

## Принципы
- Каждый артефакт сопровождается текстовой подписью «что это / из чего / для чего».
- Парные статистики — только на пересечении субъектов, `n_paired` указывается.
- Все случайные сиды и версии пакетов фиксируются в `results/final/ENV.md`.
